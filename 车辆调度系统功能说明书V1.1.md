## 1 文档概述

### 1.1 编写目的

本文档为「车辆调度系统」的功能说明书，旨在明确系统需求、业务规则、功能边界及交互细节，为后续系统设计、开发与测试提供依据。

### 1.2 适用范围

本系统面向政府部门大型活动车辆调度管理场景，以活动为核心组织维度，覆盖活动管理、车辆与司机资源管理、任务调度与执行、调度员移动端任务看板、司机端任务看板等核心业务流程。所有用车任务均在活动上下文中创建与管理。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| 活动 | 一次大型公务活动的总称，是任务的容器与上下文 |
| 任务 | 活动下的一次具体用车安排，如"接送机""办事出行"等 |
| 调度员 | 政府车辆管理人员，负责需求受理、资源调配与任务派发 |
| 司机 | 车辆驾驶人员，负责接收并执行调度任务 |
| 供应商 | 提供车辆或司机资源的外部合作单位 |
| 资源池 | 所有可用车辆/司机的集合，按状态动态更新 |

## 2 需求分析

### 2.1 用户场景分析

**场景一：活动筹备 — 资源预约与关联**
> 某部门将举办大型会议，调度员提前创建活动，录入活动名称、时间、地点等信息，从车辆池和司机池中筛选可用资源并关联至活动，为后续任务分配锁定可用资源。

**场景二：活动执行 — 任务创建与派发**
> 活动进入执行阶段，调度员在活动下逐条创建用车任务（接送机、办事出行、会议通勤等），为每个任务分配车辆和司机，并派发任务。司机通过手机端收到通知，确认接收。

**场景三：任务执行与跟踪**
> 司机在任务看板中查看待处理任务，到达任务时间后点击"开始执行"，任务完成后点击"完成任务"。调度员在管理端实时查看任务状态和执行进度。

**场景四：突发变更 — 任务改派**
> 某司机临时请假无法执行任务，调度员将任务改派给活动关联的其他空闲司机，原司机状态恢复为"可调配"。

**场景五：任务完成与资源回收**
> 任务完成后，系统自动将车辆和司机状态更新为"可调配"，资源回到活动关联池中等待下次调度，直至活动结束。

**场景六：活动收尾 — 历史查询与统计**
> 活动结束后，调度员查询本次活动下所有任务执行情况，统计车辆使用率、司机出勤率等数据，活动标记为"已结束"。

**场景七：活动现场 — 移动端调度**
> 活动进行中，调度员不在办公桌前，需要用手机随时查看当前任务执行状态，快速处理司机拒单、临时新增任务、紧急改派等现场调度需求。

### 2.2 核心需求提炼与优先级

| 优先级 | 需求 | 说明 |
|--------|------|------|
| P0 | 活动管理 | 创建活动、关联车辆与司机，活动是任务的组织容器和上下文 |
| P0 | 任务创建与派发 | 在活动下创建用车任务并派发给指定司机 |
| P0 | 司机端任务接收与执行 | 司机能查看、接收、执行、完成任务 |
| P0 | 车辆/司机资源池管理 | 维护车辆和司机基础信息及可用状态，含供应商管理 |
| P0 | 任务状态流转与跟踪 | 任务从创建到完成全流程可追踪 |
| P0 | 通知消息机制 | 任务派发、状态变更时通知相关人员 |
| P1 | 任务改派 | 司机不可用时支持重新分配 |
| P1 | 任务取消与异常处理 | 任务取消、司机拒绝等异常场景处理 |
| P1 | 调度员移动端任务看板 | 调度员在移动端查看任务看板、进行任务管理，活动场景下随时调度 |
| P2 | 快捷分配 | 系统根据任务信息智能推荐车辆和司机 |
| P2 | 车辆位置跟踪 | 任务执行过程中查看车辆实时位置 |
| P2 | 任务流水与统计查询 | 历史任务查询、活动维度统计报表 |

### 2.3 关键业务规则

1. **任务必须归属活动**：所有用车任务必须在活动下创建，不存在脱离活动的独立任务。活动是任务的组织容器和调度上下文。
2. **车辆/司机分配以时间段冲突校验为准**：同一车辆或司机在同一时间段内只能被分配一个任务，但不同时间段可复用同一车辆或司机。"已调度"状态不锁死资源，系统以任务时间段是否重叠作为唯一判据。
3. **任务状态单向流转**：任务状态只能按预设方向流转，不可逆向（取消除外）。
4. **活动与任务一对多**：一个活动可包含多个任务，一个任务只属于一个活动。
5. **供应商与资源一对多**：一个供应商可提供多辆车和多名司机。
6. **资源状态联动**：任务派发时，关联的车辆和司机状态自动更新；任务完成或取消时，自动回收到资源池。
7. **司机拒绝限制**：司机对待接收的任务可拒绝，但需填写拒绝原因，调度员收到通知后进行改派。
8. **任务超时提醒**：任务派发后超过设定时间（如30分钟）未被接收，系统自动提醒调度员。
9. **驾照匹配规则**：司机驾照类型必须≥车辆适用驾照要求。驾照等级从高到低为A1 > A2 > A3 > B1 > B2 > C1 > C2，高等级驾照可驾驶低等级要求的车辆。
10. **可用日期校验**：车辆/司机设有可用日期区间的，任务日期必须在此范围内才可分配。

## 3 系统整体设计

### 3.1 角色与权限

| 角色 | 权限范围 |
|------|----------|
| 系统管理员 | 全部功能，包括用户管理、系统配置 |
| 调度员 | 活动管理、车辆管理、司机管理、任务管理（创建/派发/改派/取消）、调度员移动端任务看板、任务流水查询、统计 |
| 司机 | 司机端任务看板（查看/接收/开始/完成/拒绝任务）、个人信息查看 |
| 供应商联系人（P2） | 查看本供应商下的车辆和司机状态 |

### 3.2 任务状态机

```
待派发 → 待接收 → 已接收 → 执行中 → 已完成
  ↓        ↓        ↓
已取消  已取消   已取消

待接收 → 已拒绝（司机拒绝）→ 待派发（改派后重新派发）
```

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待派发 | 任务已创建，未指定司机或未派发 | 编辑、派发、取消 |
| 待接收 | 任务待接收给司机，等待接收 | 取消、改派 |
| 已接收 | 司机已确认接收任务（点击"接收"或直接"开始执行"均自动标记） | 取消（调度员）、开始执行 |
| 执行中 | 司机正在执行任务 | — |
| 已完成 | 任务执行完毕 | — |
| 已取消 | 任务被取消 | — |
| 已拒绝 | 司机拒绝任务 | 改派、取消 |

> **设计说明**：保留"已接收"状态，但降低操作门槛。司机可选择"接收"确认任务，也可以直接"开始执行"（系统自动标记为已接收→执行中）。"待接收"状态下司机也可选择"拒绝"。若司机超时未操作（默认2小时），系统自动提醒调度员跟进。

### 3.3 车辆状态机

```
可调配 → 已调度 → 执行中 → 可调配（任务完成后自动回收）
  ↓
不可用（手动设置）
```

| 状态 | 说明 |
|------|------|
| 可调配 | 当前无进行中任务，可被分配新任务 |
| 已调度 | 已分配到至少一个任务，等待执行。**已调度不等于不可分配**，只要新任务与已有任务时间段不冲突，仍可继续分配 |
| 执行中 | 正在执行任务 |
| 不可用 | 维修、停用等原因暂不可调配 |

> **设计说明**：车辆/司机的可分配性以**时间段冲突校验**为准，而非简单的状态锁。同一辆车上午分配"接机任务"、下午分配"送站任务"是允许的。列表中展示"当前任务数"便于调度员快速判断资源占用情况。

### 3.4 司机状态机

```
可调配 → 已调度 → 执行中 → 可调配（任务完成后自动回收）
  ↓
不可用（手动设置）
```

| 状态 | 说明 |
|------|------|
| 可调配 | 当前无进行中任务，可被分配新任务 |
| 已调度 | 已分配到至少一个任务，等待执行。**已调度不等于不可分配**，只要新任务与已有任务时间段不冲突，仍可继续分配 |
| 执行中 | 正在执行任务 |
| 不可用 | 请假、停用等原因暂不可调配 |

### 3.5 核心业务流程

**正常调度流程：**
1. 调度员创建活动，录入活动名称、时间、地点等信息
2. 调度员将可用车辆和司机关联至活动
3. 调度员在活动下创建任务，填写任务信息（时间、地点、内容等）
4. 调度员为任务分配车辆和司机（从活动关联的车辆/司机中选择）
5. 调度员派发任务 → 任务状态变为"待接收"，系统通知司机
6. 司机接收任务 → 任务状态变为"已接收"
7. 司机开始执行 → 任务状态变为"执行中"，车辆状态同步变为"执行中"
8. 司机完成任务 → 任务状态变为"已完成"，车辆和司机自动回收到活动关联池

**改派流程：**
1. 调度员在任务详情页点击"改派"
2. 系统将原司机状态恢复为"可调配"，原车辆状态恢复为"可调配"
3. 调度员重新选择车辆和司机
4. 派发任务 → 新司机收到通知

**取消流程：**
1. 调度员在任务详情页点击"取消"
2. 填写取消原因（必填）
3. 系统将任务状态更新为"已取消"，关联车辆和司机恢复为"可调配"
4. 通知司机任务已取消

**移动端调度流程：**
1. 调度员在活动现场打开移动端，进入任务看板
2. 查看当前活动下各状态任务分布，重点关注"异常"分组
3. 发现司机拒单 → 点击任务卡片 → 长按选择"改派"→ 快捷选择替代司机 → 确认派发
4. 接到临时用车需求 → 点击"+"快速创建任务 → 智能推荐车辆+司机 → 一键分配派发
5. 收到任务超时推送 → 点击通知跳转任务详情 → 电话联系司机或改派

## 4 功能模块详细说明

### 4.1 活动管理（PC端）

#### 4.1.1 模块概述

活动是任务的容器与上下文，用于将同一批次/同一事件的用车任务组织在一起。调度员可在活动维度下统一管理车辆、司机和任务。

#### 4.1.2 功能清单

| 功能 | 说明 |
|------|------|
| 创建活动 | 新增一个活动，录入基本信息 |
| 编辑活动 | 修改活动信息 |
| 关联车辆 | 将车辆池中的可用车辆关联至活动 |
| 关联司机 | 将司机池中的可用司机关联至活动 |
| 取消关联 | 将已关联的车辆/司机从活动中移除 |
| 查看活动详情 | 查看活动信息及关联的任务、车辆、司机 |
| 结束活动 | 活动结束后标记为"已结束"，关联的未完成任务需先处理 |
| 活动列表 | 按状态/时间筛选查看所有活动 |

#### 4.1.3 功能详细说明

**创建活动**

- 操作入口：活动管理页 → 新建活动
- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 活动名称 | 文本 | 是 | 如"2026年全市工作会议" |
| 活动时间 | 时间范围 | 是 | 开始时间~结束时间 |
| 活动地点 | 文本 | 否 | 活动举办地点 |
| 活动描述 | 富文本 | 否 | 活动背景、特殊要求等 |
| 负责人 | 选择 | 否 | 关联系统中的调度员 |
| 状态 | 枚举 | 自动 | 默认"筹备中" |

- 活动状态：筹备中 → 进行中 → 已结束
- 交互逻辑：填写表单 → 点击保存 → 系统创建活动，跳转至活动详情页
- 边界条件：活动时间不可早于当前时间（创建时校验）；同一时间段允许存在多个活动

**关联车辆/司机**

- 操作入口：活动详情页 → 关联车辆/关联司机
- 交互逻辑：弹窗显示当前资源池中"可调配"状态的车辆/司机列表 → 勾选后确认 → 关联成功，车辆/司机状态不变（仍为"可调配"，仅建立关联关系）
- 边界条件：同一车辆/司机可被关联至多个活动（因不同活动时间段可能不冲突）；已被关联但状态正常的资源可重复关联

**取消关联**

- 交互逻辑：在活动详情页点击"取消关联"→ 确认 → 移除关联
- 边界条件：若该车辆/司机已被分配到本活动下的某个任务，不可取消关联（需先改派或取消任务）

**结束活动**

- 交互逻辑：点击"结束活动"→ 系统校验是否还有"待派发/待接收/已接收/执行中"状态的任务 → 若有则提示需先处理 → 若无则标记为"已结束"
- 边界条件：活动结束后，关联的空闲车辆/司机自动取消关联，回到资源池

### 4.2 车辆管理池（PC端）

#### 4.2.1 模块概述

集中管理所有可用车辆信息及供应商信息，为活动关联和任务分配提供车辆资源。

#### 4.2.2 功能清单

| 功能 | 说明 |
|------|------|
| 新增车辆 | 录入车辆基础信息 |
| 编辑车辆 | 修改车辆信息 |
| 删除车辆 | 删除无效车辆记录 |
| 设置不可用 | 标记车辆为维修/停用等不可调配状态 |
| 恢复可用 | 将不可用车辆恢复为可调配 |
| 车辆列表 | 按状态/供应商/类型筛选查看 |


#### 4.2.3 功能详细说明

**新增车辆**

- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 车牌号 | 文本 | 是 | 唯一标识，系统校验不可重复 |
| 车辆类型 | 枚举 | 是 | 轿车/SUV/商务车/中巴/大巴 |
| 品牌型号 | 文本 | 否 | 如"别克GL8" |
| 核载人数 | 数字 | 否 | 不含司机 |
| 颜色 | 文本 | 否 | — |
| 适用驾照 | 枚举 | 否 | 驾驶该车辆所需的最低驾照类型，如C1/B1/A1。分配司机时系统校验司机驾照类型是否≥此要求 |
| 可用日期区间 | 时间范围 | 否 | 车辆可被调配的日期范围（如租期2026-06-01~2026-06-30），不填则视为长期可用；分配任务时校验任务日期是否在此范围内 |
| 所属供应商 | 选择 | 否 | 关联供应商管理中的记录 |
| 联系电话 | 文本 | 否 | 车辆负责人/供应商联系方式 |
| 备注 | 文本 | 否 | — |
| 状态 | 枚举 | 自动 | 默认"可调配" |

- 交互逻辑：填写表单 → 保存 → 系统校验车牌号唯一性 → 创建成功
- 边界条件：车牌号重复时提示错误；新增后自动进入资源池；可用日期区间支持多段维护（如车辆租期分两段）

**设置不可用**

- 交互逻辑：点击"设为不可用"→ 选择原因（维修/年检/停用/其他）→ 确认 → 状态变为"不可用"
- 边界条件：若车辆当前状态为"已调度"或"执行中"，不可设为不可用（需先处理关联任务）

**删除车辆**

- 交互逻辑：点击"删除"→ 确认 → 删除记录
- 边界条件：状态非"可调配"和"不可用"时不可删除；建议优先使用"设为不可用"而非删除

### 4.3 司机管理池（PC端）

#### 4.3.1 模块概述

集中管理所有可用司机信息及供应商信息，为活动关联和任务分配提供司机资源。

#### 4.3.2 功能清单

| 功能 | 说明 |
|------|------|
| 新增司机 | 录入司机基础信息 |
| 编辑司机 | 修改司机信息 |
| 删除司机 | 删除无效司机记录 |
| 设置不可用 | 标记司机为请假/停用等不可调配状态 |
| 恢复可用 | 将不可用司机恢复为可调配 |
| 司机列表 | 按状态/供应商筛选查看 |


#### 4.3.3 功能详细说明

**新增司机**

- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | 是 | — |
| 手机号 | 文本 | 是 | 用于登录和接收通知，唯一 |
| 身份证号 | 文本 | 否 | — |
| 驾照类型 | 枚举 | 否 | C1/B1/A1等。分配任务时系统校验司机驾照类型是否≥车辆适用驾照要求 |
| 可用日期区间 | 时间范围 | 否 | 司机可被调配的日期范围（如聘用期2026-06-01~2026-06-30），不填则视为长期可用；分配任务时校验任务日期是否在此范围内 |
| 所属供应商 | 选择 | 否 | 关联供应商管理中的记录 |
| 紧急联系人 | 文本 | 否 | — |
| 紧急联系电话 | 文本 | 否 | — |
| 备注 | 文本 | 否 | — |
| 状态 | 枚举 | 自动 | 默认"可调配" |

- 交互逻辑：填写表单 → 保存 → 系统校验手机号唯一性 → 创建成功，同时创建司机端登录账号
- 边界条件：手机号重复时提示错误；可用日期区间支持多段维护

**设置不可用**

- 交互逻辑：点击"设为不可用"→ 选择原因（请假/停用/其他）+ 预计恢复时间（选填）→ 确认 → 状态变为"不可用"
- 边界条件：若司机当前状态为"已调度"或"执行中"，不可设为不可用

**删除司机**

- 边界条件：状态非"可调配"和"不可用"时不可删除；有历史任务记录的司机建议设为不可用而非删除

### 4.4 任务管理（PC端）

#### 4.4.1 模块概述

任务是车辆调度的核心业务单元，调度员在此模块创建、分配、派发和跟踪所有用车任务。

#### 4.4.2 功能清单

| 功能 | 说明 |
|------|------|
| 创建任务 | 在活动下新建用车任务 |
| 编辑任务 | 修改未派发的任务信息 |
| 分配车辆与司机 | 为任务指定车辆和司机 |
| 派发任务 | 将任务派发给司机 |
| 改派任务 | 更换任务的车辆或司机 |
| 取消任务 | 取消已创建的任务 |
| 任务列表 | 按活动/状态/时间筛选查看 |
| 任务详情 | 查看任务完整信息和状态变更记录 |
| 任务跟踪 | 实时查看执行中任务的状态 |

#### 4.4.3 功能详细说明

**创建任务**

- 操作入口：活动详情页 → 新建任务
- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属活动 | 自动 | 是 | 从活动详情页创建时自动继承，不可修改 |
| 任务名称 | 文本 | 是 | 如"6月10日张局长接机" |
| 任务类型 | 枚举 | 是 | 接送机/办事出行/会议通勤/考察调研/其他 |
| 任务日期 | 日期 | 是 | — |
| 开始时间 | 时间 | 是 | 用车开始时间 |
| 结束时间 | 时间 | 是 | 预计用车结束时间 |
| 出发地 | 文本 | 是 | — |
| 目的地 | 文本 | 是 | — |
| 乘车人/接口人 | 文本 | 否 | — |
| 乘车人电话 | 文本 | 否 | — |
| 乘车人数 | 数字 | 否 | 用于匹配车辆座位数 |
| 任务描述 | 富文本 | 否 | 特殊要求、路线说明等 |
| 分配车辆 | 选择 | 否 | 创建时可暂不分配 |
| 分配司机 | 选择 | 否 | 创建时可暂不分配 |
| 状态 | 枚举 | 自动 | 默认"待派发" |

- 交互逻辑：填写表单 → 保存 → 任务创建成功，状态为"待派发"
- 边界条件：任务必须在活动下创建，不可脱离活动独立存在；任务时间应在活动时间范围内（超出范围时系统提示警告但允许保存，因实际调度可能需要提前出发）；创建时若未分配车辆/司机，状态为"待派发"，需分配后再派发

**分配车辆与司机**

- 操作入口：任务详情页 → 分配车辆/分配司机
- 交互逻辑：
  1. 点击"分配车辆"→ 弹窗显示本活动关联的车辆列表，优先展示时间段不冲突的车辆，标注每辆车的当前任务数、可用日期区间、适用驾照 → 选择车辆 → 确认
  2. 点击"分配司机"→ 弹窗显示本活动关联的司机列表，优先展示时间段不冲突的司机，标注每位司机的当前任务数、可用日期区间、驾照类型 → 选择司机 → 确认
  3. 分配后，车辆和司机状态变为"已调度"
- 校验规则（分配时系统自动校验，不通过则阻止分配并提示原因）：
  - **时间段冲突校验**：新任务的时间段与该车辆/司机已有任务的时间段不可重叠
  - **可用日期校验**：任务日期必须在车辆/司机的可用日期区间内（未设置可用日期区间的跳过此校验）
  - **驾照匹配校验**：司机的驾照类型必须≥车辆适用驾照要求（如车辆要求B1，司机需持有B1或A1，C1不可匹配）
  - **供应商状态校验**：若车辆/司机所属供应商状态为"已终止"，不可分配；为"已暂停"时提示警告但允许分配
- 边界条件：
  - 若活动未关联车辆/司机，显示全局资源池中的可调配车辆/司机
  - 可选"快捷分配"：系统根据任务时间、座位需求、驾照匹配等条件自动推荐最匹配的车辆和司机
  - 乘车人数超过车辆核载人数时，系统提示警告

**派发任务**

- 操作入口：任务详情页 → 派发
- 前置条件：任务已分配车辆和司机
- 交互逻辑：点击"派发"→ 确认 → 任务状态变为"待接收"→ 系统向司机发送通知（App推送/短信）
- 边界条件：未分配车辆或司机时不可派发

**改派任务**

- 操作入口：任务详情页 → 改派（仅"待接收""已接收""已拒绝"状态可改派）
- 交互逻辑：
  1. 点击"改派"→ 选择改派类型：仅换司机/仅换车辆/同时更换
  2. 系统将原车辆/司机状态恢复为"可调配"
  3. 重新选择车辆/司机 → 确认
  4. 新司机收到任务通知，原司机收到改派通知
  5. 任务状态变为"待接收"
- 边界条件：改派后原司机若有疑问可查看改派记录

**取消任务**

- 操作入口：任务详情页 → 取消（仅"待派发""待接收""已接收"状态可取消）
- 字段：取消原因（必填）
- 交互逻辑：填写原因 → 确认 → 任务状态变为"已取消"→ 关联车辆/司机恢复为"可调配"→ 通知司机
- 边界条件："执行中"状态的任务不可直接取消，需联系司机确认后由系统管理员操作

**任务跟踪**

- 管理端展示当前所有"待接收""已接收""执行中"的任务列表
- 每条任务显示：任务名称、司机、车辆、当前状态、计划时间
- 支持按状态筛选，支持刷新
- P2：执行中的任务可查看车辆实时位置（地图展示）

### 4.5 司机端 - 任务看板（移动端）

#### 4.5.1 模块概述

司机通过移动端查看和管理自己的任务，是任务执行的唯一入口。

#### 4.5.2 功能清单

| 功能 | 说明 |
|------|------|
| 任务列表 | 查看当前待处理和进行中的任务 |
| 任务详情 | 查看任务完整信息 |
| 接收任务 | 确认接收待接收的任务 |
| 拒绝任务 | 拒绝待接收的任务（需填原因） |
| 开始执行 | 开始执行已接收的任务 |
| 完成任务 | 标记任务为已完成 |
| 历史任务 | 查看已完成/已取消的历史任务 |

#### 4.5.3 功能详细说明

**任务列表**

- 展示字段：任务名称、任务类型、任务日期、开始时间、出发地→目的地、状态
- 筛选：按状态筛选（待接收/进行中/已完成）
- 排序：按任务时间正序排列，最近的任务在前
- 交互逻辑：司机登录后默认进入任务列表页，下拉刷新

**接收任务**

- 操作入口：任务详情页 → 接收任务
- 前置条件：任务状态为"待接收"
- 交互逻辑：点击"接收"→ 确认 → 任务状态变为"已接收"
- 说明：接收为非必须步骤，司机也可直接点击"开始执行"（系统自动标记为已接收→执行中）。接收的目的是让调度员知道司机已知晓任务。若司机2小时内既未接收也未拒绝，系统自动提醒调度员跟进

**拒绝任务**

- 操作入口：任务详情页 → 拒绝任务
- 前置条件：任务状态为"待接收"
- 字段：拒绝原因（必填）
- 交互逻辑：填写原因 → 确认 → 任务状态变为"已拒绝"→ 通知调度员 → 调度员进行改派
- 边界条件：司机只能在"待接收"状态下拒绝；"已接收"后不可拒绝，需联系调度员处理

**开始执行**

- 操作入口：任务详情页 → 开始执行
- 前置条件：任务状态为"待接收"或"已接收"
- 交互逻辑：点击"开始执行"→ 确认 → 若当前状态为"待接收"则自动流转为"已接收→执行中"；若为"已接收"则直接变为"执行中"；关联车辆状态同步变为"执行中"
- 边界条件：到达计划开始时间前30分钟才可点击"开始执行"（提前点击提示"尚未到任务执行时间"）；也可允许司机提前开始，但需二次确认

**完成任务**

- 操作入口：任务详情页 → 完成任务
- 前置条件：任务状态为"执行中"
- 交互逻辑：点击"完成"→ 确认 → 任务状态变为"已完成"，关联车辆和司机状态恢复为"可调配"
- 边界条件：任务完成后司机可在详情页补充任务备注（如实际到达时间、特殊路况等）

**历史任务**

- 展示已完成和已取消的任务列表
- 支持按日期范围筛选
- 不可对历史任务进行操作

### 4.6 调度员移动端 - 任务看板与任务管理（移动端）

#### 4.6.1 模块概述

调度员在活动现场或外出时，通过移动端查看任务执行状态、处理异常任务、快速创建和派发任务。与PC端任务管理功能互补：移动端侧重**实时监控与快速响应**，PC端侧重**批量操作与详细编辑**。

#### 4.6.2 设计原则

1. **移动端与PC端数据完全同步**：所有操作实时生效，不存在两端数据不一致
2. **移动端聚焦高频操作**：优先支持任务看板、任务派发、改派、取消、状态查看等现场高频场景
3. **简化操作路径**：核心操作控制在2~3步内完成，减少移动端输入负担
4. **活动/司机资源管理等低频复杂操作**：移动端仅提供查看入口，编辑操作引导至PC端完成

#### 4.6.3 功能清单

| 功能 | 说明 | 与PC端差异 |
|------|------|------------|
| 任务看板 | 按状态分组展示当前活动下所有任务 | 移动端独有，PC端为列表模式 |
| 任务列表 | 按活动/状态/时间筛选查看任务 | 筛选条件简化，列表字段精简 |
| 任务详情 | 查看任务完整信息及状态变更记录 | 与PC端一致 |
| 快速创建任务 | 在活动下快速创建用车任务 | 字段精简，支持语音/模板创建 |
| 分配与派发 | 为任务分配车辆/司机并派发 | 操作流程简化，支持快捷分配 |
| 改派任务 | 更换任务的车辆或司机 | 与PC端一致 |
| 取消任务 | 取消已创建的任务 | 与PC端一致 |
| 活动选择与切换 | 切换当前查看的活动上下文 | 移动端独有快捷入口 |
| 实时状态推送 | 任务状态变更时推送通知 | 移动端独有通知栏 |

#### 4.6.4 功能详细说明

**任务看板**

- 操作入口：调度员移动端首页
- 展示逻辑：
  - 顶部显示当前活动选择器，可快速切换活动
  - 按任务状态分组展示，形成看板视图：
    - **待派发**：待分配/待派发的任务，显示任务名称、时间、车辆/司机分配状态
    - **进行中**：待接收/已接收/执行中的任务，显示司机姓名、车辆、当前状态标签
    - **异常**：已拒绝/超时未接收的任务，红色标记提醒，显示异常原因
    - **已完成**：今日已完成任务数汇总（折叠展示）
  - 每个状态分组显示任务数量角标
- 交互逻辑：
  - 点击任务卡片 → 进入任务详情页
  - 长按任务卡片 → 弹出快捷操作菜单（派发/改派/取消）
  - 下拉刷新更新任务状态
  - 看板数据自动轮询刷新（间隔可配置，默认30秒）
- 边界条件：未关联活动时看板为空，引导用户先在PC端创建活动并关联资源

**任务列表**

- 展示字段：任务名称、任务类型标签、开始时间、司机姓名、车辆车牌号、状态标签
- 筛选：按状态筛选（全部/待派发/进行中/异常/已完成）、按日期筛选（今日/本周/自定义）
- 排序：按任务开始时间正序
- 交互逻辑：点击任务 → 进入任务详情

**任务详情**

- 展示内容：
  - 任务基本信息：名称、类型、时间、出发地→目的地、乘车人、乘车人电话、任务描述
  - 分配信息：车辆（车牌号、类型、供应商）、司机（姓名、电话、驾照类型、供应商）
  - 状态变更时间线：记录每次状态变更的时间、操作人、备注
  - 操作按钮：根据当前状态动态显示（派发/改派/取消）
- 交互逻辑：
  - 点击司机电话 → 直接拨号
  - 点击乘车人电话 → 直接拨号
  - 操作按钮点击后执行对应功能

**快速创建任务**

- 操作入口：任务看板页 → 右下角"+"按钮
- 前置条件：已选择当前活动
- 字段（精简版，对比PC端减少非必要字段）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属活动 | 自动 | 是 | 自动继承当前看板所选活动，可切换 |
| 任务名称 | 文本 | 是 | — |
| 任务类型 | 枚举 | 是 | 接送机/办事出行/会议通勤/考察调研/其他 |
| 任务日期 | 日期 | 是 | 默认当天 |
| 开始时间 | 时间 | 是 | — |
| 结束时间 | 时间 | 是 | — |
| 出发地 | 文本 | 是 | — |
| 目的地 | 文本 | 是 | — |
| 乘车人/接口人 | 文本 | 否 | — |
| 乘车人电话 | 文本 | 否 | — |
| 乘车人数 | 数字 | 否 | — |
| 任务描述 | 文本 | 否 | 简化为纯文本（非富文本） |
| 分配车辆 | 选择 | 否 | 可创建后再分配 |
| 分配司机 | 选择 | 否 | 可创建后再分配 |

- 交互逻辑：填写表单 → 点击"创建"→ 任务创建成功 → 提示"是否立即分配并派发？"→ 是则进入分配流程，否则返回看板
- 边界条件：移动端创建的任务与PC端完全一致，数据互通；移动端暂不支持富文本编辑，任务描述为纯文本

**分配与派发**

- 操作入口：任务详情页 → 分配车辆/分配司机 → 派发
- 交互逻辑（简化流程）：
  1. 点击"分配车辆"→ 列表展示活动关联的可用车辆，顶部标注推荐项（快捷分配）→ 选择 → 确认
  2. 点击"分配司机"→ 列表展示活动关联的可用司机，顶部标注推荐项 → 选择 → 确认
  3. 点击"派发"→ 确认 → 任务派发，通知司机
- 校验规则：与PC端一致（时间段冲突、可用日期、驾照匹配、供应商状态）
- 快捷分配：点击"智能推荐"→ 系统自动推荐最匹配的车辆+司机组合 → 一键确认分配并派发
- 边界条件：移动端分配校验规则与PC端完全一致，不因端不同而放松校验

**改派任务**

- 操作入口：任务详情页 → 改派（仅"待接收""已接收""已拒绝"状态可改派）
- 交互逻辑：与PC端一致，选择改派类型 → 重新选择资源 → 确认 → 通知新旧司机
- 边界条件：与PC端一致

**取消任务**

- 操作入口：任务详情页 → 取消（仅"待派发""待接收""已接收"状态可取消）
- 交互逻辑：与PC端一致，填写取消原因 → 确认 → 通知司机
- 边界条件：与PC端一致

**活动选择与切换**

- 操作入口：任务看板顶部 → 活动选择器
- 交互逻辑：
  - 显示当前进行中的活动列表（筹备中+进行中状态）
  - 选择活动后，看板和列表自动切换为该活动下的任务
  - 支持"全部活动"视图，汇总所有活动的待处理任务
- 边界条件：活动的创建、编辑、资源关联等管理操作引导至PC端完成

**实时状态推送**

- 调度员移动端接收以下推送通知（与第5章通知机制联动）：
  - 司机接收/拒绝任务
