## 1 文档概述

### 1.1 编写目的

本文档为「车辆调度系统」的功能说明书，旨在明确系统需求、业务规则、功能边界及交互细节，为后续系统设计、开发与测试提供依据。

### 1.2 适用范围

本系统面向政府部门大型活动车辆调度管理场景，以活动为核心组织维度，覆盖活动管理、供应商管理、车辆与司机资源管理（含审核流程）、任务调度与执行、轨迹追踪与轨迹大屏、异常任务识别与处理、调度员移动端、司机端等核心业务流程。所有用车任务均在活动上下文中创建与管理。活动需先关联供应商，车辆与司机需经活动负责人审核通过后方可进入可分配资源池。活动结束后关联车辆和司机自动置为不可用状态。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| 活动 | 一次大型公务活动的总称，是任务的容器与上下文 |
| 任务 | 活动下的一次具体用车安排，如"接送机""办事出行"等 |
| 活动总调度员 | 活动负责人，可查看和操作活动下所有任务，一般为活动创建者或被指定的负责人 |
| 现场调度员 | 负责具体任务执行跟踪的调度人员，每个任务指定一名现场调度员，默认为任务创建者，只能查看和操作分配给自己的任务 |
| 司机 | 车辆驾驶人员，负责接收并执行调度任务 |
| 供应商 | 提供车辆或司机资源的外部合作单位，分为车辆供应商、司机供应商和综合供应商 |
| 供应商联系人 | 供应商方的系统操作人员，负责录入本供应商名下的车辆和司机信息及审核材料 |
| 活动关联供应商 | 活动与供应商的关联关系，活动需先关联供应商后，该供应商名下的车辆和司机才可被添加到本活动 |
| 途经点 | 任务路线中出发地与目的地之间的中间停靠点，一个任务可添加多个途经点，构成完整行驶路线 |
| 工作小组 | 活动下按地点划分的工作单元，每个工作小组维护地点、现场负责人和现场调度员，用于任务审批和权限控制 |
| 现场负责人 | 工作小组中负责审批任务的现场管理人员，非必填。现场调度员新建任务时，若任务地点对应的工作小组有现场负责人，则由其审批；若无，则由活动总调度员审批 |
| 任务审批 | 现场调度员新建任务后，需经现场负责人或活动总调度员审批通过方可分配车辆和司机 |
| 暂停任务 | 司机执行中因故暂停任务，需填写暂停原因，经总调度员审核通过后暂停GPS采集 |
| 故障上报 | 司机对已接收/执行中的任务上报车辆故障，上报后任务标记为异常，车辆置为不可用 |
| 催办 | 对待接收状态的任务，调度员主动发送站内通知提醒司机及时接收 |
| 资源池 | 所有审核通过且可用的车辆/司机的集合，按状态动态更新 |
| 轨迹大屏 | PC端全局监控视图，以地图形式展示活动下所有执行中司机的实时位置，支持活动切换与司机详情下钻 |
| 异常任务 | 不满足正常执行状态的任务，系统按预设规则自动识别并标记，便于调度员快速定位和处理 |
| 筹备期 | 活动创建后的初始阶段，可管理资源和创建任务，但不可派发任务 |
| 执行期 | 活动负责人点击"开始活动"后进入的阶段，可进行任务派发与调度 |
| 结束期 | 活动收尾阶段，所有任务办结后由活动负责人标记结束，关联车辆/司机自动置为不可用 |

## 2 需求分析

### 2.1 用户场景分析

**场景一：活动筹备 — 关联供应商与资源录入**
> 某部门将举办大型会议，调度员提前创建活动（进入筹备期），指定活动负责人。调度员在活动详情页"关联供应商"Tab中，从供应商池中选择合作中的供应商关联至本次活动。供应商联系人收到关联通知后，登录系统录入本供应商名下的车辆和司机信息，上传审核材料，提交审核。由于活动已关联该供应商，车辆/司机新增时的活动选择列表仅展示本供应商关联的筹备期/执行期活动。活动负责人审核通过后，车辆和司机进入活动关联资源池。调度员在活动下批量创建用车任务，为任务分配车辆和司机（筹备期不派发，仅预分配）。

**场景二：活动启动 — 任务派发**
> 活动负责人确认资源就绪后，点击"开始活动"，活动从筹备期进入执行期。调度员开始批量派发已预分配的任务，司机通过手机端收到通知，确认接收并执行。

**场景三：任务执行与轨迹追踪**
> 司机开始执行任务后，系统实时记录司机位置与行动轨迹。调度员在任务详情页查看司机当前定位和已行驶轨迹，在轨迹大屏上查看活动下所有执行中司机的位置分布，点击司机可查看其任务详情。

**场景四：突发变更 — 任务改派**
> 某司机临时请假无法执行任务，系统自动将任务标记为"异常"（司机拒绝未改派），调度员收到异常提醒，将任务改派给活动关联的其他空闲司机，原司机状态恢复为"可调配"。

**场景五：异常识别与处理**
> 任务到达计划开始时间但司机尚未开始执行，系统自动标记为"异常任务"并在任务列表和活动信息页突出展示。调度员快速定位异常，电话联系司机或改派。

**场景六：活动收尾 — 资源回收与不可用**
> 活动结束后，系统自动将本次活动关联的所有车辆和司机状态置为"不可用"，防止资源在未确认可用的情况下被分配到其他活动。后续如需复用，调度员手动将对应车辆/司机恢复为"可调配"状态。调度员查询本次活动下所有任务执行情况、轨迹回放、异常处理记录，统计车辆使用率、司机出勤率等数据。

**场景七：供应商管理 — 资源来源管控**
> 调度员新增一家合作供应商，录入资质信息与联系方式，为供应商联系人开通系统账号。在活动详情页关联该供应商后，供应商联系人登录系统，录入本公司车辆信息并上传车辆照片、年检合格证、保险单等审核材料，录入司机信息并上传驾驶证等材料，提交审核。活动负责人审核通过后，车辆和司机进入资源池。当某供应商合作到期或违约时，调度员将供应商状态置为"已终止"，系统自动阻止该供应商下所有车辆和司机参与新任务分配。

**场景八：供应商参与活动查看**
> 调度员打开某供应商详情页，"参与活动"区域列出该供应商已关联的所有活动（含活动名称、时期、关联时间）。在名下车辆和名下司机Tab中，"所属活动"列展示每辆车/每名司机所关联的活动名称，方便调度员快速了解资源分布。

**场景九：轨迹大屏 — 全局监控**
> 活动进行中，总调度员打开PC端轨迹大屏，切换至目标活动，地图上展示所有执行中司机的实时位置和方向。发现某司机偏离路线，点击该司机图标查看其任务详情和已行驶轨迹，及时电话联系确认。

**场景十：调度员分级管理**
> 大型活动有多名现场调度员各负责一部分任务。张调度员创建并负责接机任务，李调度员负责送站任务，两人各自只能看到自己的任务。活动负责人（总调度员）可查看活动下所有任务，统一协调。

**场景十一：资源审核**
> 供应商联系人录入一辆别克GL8，上传车辆照片、年检合格证和保险单，提交审核。活动负责人在审核列表中看到该车辆，查看审核材料，确认无误后点击"审核通过"，车辆状态从"待审核"变为"可调配"，进入资源池。若材料不全或信息有误，活动负责人点击"审核驳回"并填写驳回原因，供应商联系人收到通知后修改信息重新提交。

**场景十二：移动端任务统计**
> 司机打开移动端，首页任务页顶部显示任务统计区（待接收3、执行中2、已完成5），点击"待接收"数字，下方展开该状态的任务列表，任务卡片紧凑展示任务名称、时间和状态，操作便捷。

### 2.3 关键业务规则

1. **任务必须归属活动**：所有用车任务必须在活动下创建，不存在脱离活动的独立任务。活动是任务的组织容器和调度上下文。
2. **车辆/司机分配以时间段冲突校验为准**：同一车辆或司机在同一时间段内只能被分配一个任务，但不同时间段可复用同一车辆或司机。"已调度"状态不锁死资源，系统以任务时间段是否重叠作为唯一判据。
3. **任务状态单向流转**：任务状态只能按预设方向流转，不可逆向（取消除外）。
4. **活动与任务一对多**：一个活动可包含多个任务，一个任务只属于一个活动。
5. **供应商与资源一对多**：一个供应商可提供多辆车和多名司机。
6. **资源状态联动**：任务派发时，关联的车辆和司机状态自动更新；任务完成或取消时，自动回收到资源池。
7. **司机拒绝限制**：司机对待接收的任务可拒绝，但需填写拒绝原因，调度员收到通知后进行改派。
8. **任务超时提醒**：任务派发后超过设定时间（如30分钟）未被接收，系统自动提醒调度员。
9. **驾照匹配规则**：司机驾照类型必须≥车辆适用驾照要求。驾照等级从高到低为A1 > A2 > A3 > B1 > B2 > C1 > C2。
10. **可用日期校验**：车辆/司机设有可用日期区间的，任务日期必须在此范围内才可分配。
11. **供应商状态联动校验**：车辆/司机分配时，系统校验其所属供应商状态。"已终止"禁止分配，"已暂停"警告后可分配。
12. **调度员分级权限**：现场调度员只能查看和操作分配给自己的任务；活动总调度员（活动负责人）可查看和操作活动下所有任务。任务创建时自动指定创建者为现场调度员，可手动修改。
13. **异常任务自动识别**：系统按预设规则自动识别异常任务并标记，详见第3.7节异常任务判断规则。
14. **资源审核准入**：车辆和司机由供应商联系人录入并上传审核材料后，须经活动负责人审核通过方可进入可分配资源池。审核不通过时供应商联系人可修改后重新提交。
15. **活动时期管控**：活动分为筹备期、执行期、结束期三个时期。筹备期可管理车辆/司机、新增任务、分配资源，但不可派发任务；执行期可进行全部调度操作；结束期活动归档。活动负责人点击"开始活动"后从筹备期进入执行期。
16. **活动必须先关联供应商**：活动需先关联供应商（从供应商池中选择状态为"合作中"的供应商），该供应商名下的车辆/司机才可被添加到本活动。车辆/司机新增时，"所属活动"下拉列表仅显示所属供应商已关联的、且处于筹备期或执行期的活动。
17. **活动结束后资源不可用**：活动结束（进入结束期）后，系统自动将本次活动关联的所有车辆和司机状态置为"不可用"。后续如需复用，需由调度员手动恢复为"可调配"状态。
18. **工作小组管理**：活动下可创建多个工作小组，每个工作小组维护地点、现场负责人（非必填）、现场调度员。工作小组是任务审批和车辆分配权限的依据。
19. **任务审批机制**：现场调度员新建任务后，任务状态为"待审批"。系统根据任务出发地匹配工作小组：若匹配到有现场负责人的工作小组，则由该现场负责人审批；若匹配到的工作小组无现场负责人，或未匹配到工作小组，则由活动总调度员审批。审批通过后任务变为"待派发"，方可分配车辆和司机；审批不通过则任务终止。活动总调度员新建的任务无需审批，直接进入"待派发"状态。
20. **车辆绑定默认司机**：审核通过的车辆可绑定已审核通过的司机作为默认司机。在用车任务中分配车辆时，系统自动带出该车辆绑定的默认司机，调度员可手动调整。
21. **车辆绑定活动地点**：车辆可绑定活动中的工作小组地点。绑定过地点的车辆，现场调度员有权限进行分配；未绑定地点的车辆，仅活动总调度员有权限分配。
22. **司机暂停/继续执行**：司机可在执行中申请暂停任务，需填写暂停原因，经总调度员审核通过后暂停GPS采集。点击"继续执行"不需审批，立即恢复GPS采集。
23. **故障上报**：司机对已接收或执行中的任务可进行故障上报。上报后任务标记为异常（E6），同时该任务关联的车辆状态置为"不可用"。
24. **催办机制**：对"待接收"状态的任务，调度员可点击"催办"按钮，系统向司机发送站内通知，提醒及时接收任务。

## 3 系统整体设计

### 3.1 角色与权限

| 角色 | 权限范围 |
|------|----------|
| 系统管理员 | 全部功能，包括用户管理、系统配置 |
| 活动总调度员 | 作为活动负责人，可查看和操作该活动下所有任务；活动管理（含关联供应商、工作小组管理）、供应商管理、车辆/司机审核、车辆管理、司机管理、轨迹大屏、任务流水查询、统计；审批无现场负责人的任务；审核司机暂停申请；分配未绑定地点的车辆 |
| 现场负责人 | 审批对应工作小组地点的任务；查看对应工作小组下的任务列表；不参与调度操作 |
| 现场调度员 | 查看和操作分配给自己的任务；调度员移动端任务管理（仅自己的任务）；车辆/司机管理（查看）、供应商管理（查看）；新建任务需审批；分配已绑定地点的车辆 |
| 司机 | 司机端任务管理（查看/接收/开始/暂停/继续执行/完成/拒绝任务、故障上报）、个人信息查看 |
| 供应商联系人 | 录入/编辑/删除本供应商名下车辆和司机信息及审核材料；查看本供应商下车辆/司机审核状态和使用状态；接收审核结果通知；查看本供应商关联的活动信息 |

> **说明**：活动总调度员和现场调度员不是独立的系统角色，而是同一"调度员"角色在不同活动中的权限表现。调度员在某活动中被指定为负责人时即为该活动的总调度员；调度员创建任务时自动成为该任务的现场调度员。一个调度员可以同时是A活动的总调度员和B活动的现场调度员。现场负责人是工作小组中的审批角色，由活动总调度员在工作小组中指定，负责审批对应地点的任务，不参与日常调度操作。供应商联系人是独立角色，绑定至特定供应商账号，仅可操作本供应商名下资源。

### 3.2 任务状态机

```
待审批 → 待派发（审批通过）
待审批 → 已取消（审批不通过）

待派发 → 待接收 → 已接收 → 执行中 → 已完成
  ↓        ↓        ↓          ↓
已取消  已取消   已取消     已暂停 → 执行中（继续执行）

待接收 → 已拒绝（司机拒绝）→ 待派发（改派后重新派发）
执行中 → 已暂停（司机申请暂停，总调度员审批通过）
已暂停 → 执行中（司机点击继续执行，不需审批）
```

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待审批 | 现场调度员新建任务后等待审批 | 取消 |
| 待派发 | 任务审批通过，未指定司机或未派发 | 编辑、分配车辆/司机、派发（仅执行期）、取消 |
| 待接收 | 任务已派发给司机，等待接收 | 改派、取消、催办 |
| 已接收 | 司机已确认接收任务 | 取消（调度员）、开始执行、故障上报 |
| 执行中 | 司机正在执行任务 | 暂停任务、故障上报 |
| 已暂停 | 司机申请暂停且总调度员审批通过 | 继续执行 |
| 已完成 | 任务执行完毕 | — |
| 已取消 | 任务被取消 | — |
| 已拒绝 | 司机拒绝任务 | 改派、取消 |

> **设计说明**：保留"已接收"状态，但降低操作门槛。司机可选择"接收"确认任务，也可以直接"开始执行"（系统自动标记为已接收→执行中）。若司机超时未操作（默认2小时），系统自动提醒调度员跟进。现场调度员新建的任务需经审批（现场负责人或活动总调度员）后方可分配车辆和司机，活动总调度员新建的任务无需审批直接进入"待派发"。

### 3.3 车辆状态机

```
待审核 → 可调配（审核通过）
  ↓         ↓
审核不通过   已调度 → 执行中 → 可调配（任务完成后自动回收）
  ↓           ↓
待审核      不可用（手动设置）
（修改后     ↑
重新提交）  可调配 → 不可用
```

| 状态 | 说明 |
|------|------|
| 待审核 | 供应商联系人已录入车辆信息及审核材料，等待活动负责人审核 |
| 审核不通过 | 活动负责人审核驳回，供应商联系人可修改后重新提交 |
| 可调配 | 审核通过且当前无进行中任务，可被分配新任务 |
| 已调度 | 已分配到至少一个任务，等待执行。**已调度不等于不可分配**，只要新任务与已有任务时间段不冲突，仍可继续分配 |
| 执行中 | 正在执行任务 |
| 不可用 | 维修、停用等原因暂不可调配；**活动结束后系统自动置为不可用** |

> **设计说明**：车辆首次创建时状态为"待审核"，只有活动负责人审核通过后才变为"可调配"进入资源池。审核不通过的车辆可修改信息后重新提交审核。已审核通过后设置为"不可用"的车辆，恢复时直接变回"可调配"，无需再次审核。活动结束后，关联的所有车辆自动置为"不可用"，需调度员手动恢复。

### 3.4 司机状态机

```
待审核 → 可调配（审核通过）
  ↓         ↓
审核不通过   已调度 → 执行中 → 可调配（任务完成后自动回收）
  ↓           ↓
待审核      不可用（手动设置）
（修改后     ↑
重新提交）  可调配 → 不可用
```

| 状态 | 说明 |
|------|------|
| 待审核 | 供应商联系人已录入司机信息及审核材料，等待活动负责人审核 |
| 审核不通过 | 活动负责人审核驳回，供应商联系人可修改后重新提交 |
| 可调配 | 审核通过且当前无进行中任务，可被分配新任务 |
| 已调度 | 已分配到至少一个任务，等待执行 |
| 执行中 | 正在执行任务 |
| 不可用 | 请假、停用等原因暂不可调配；**活动结束后系统自动置为不可用** |

> **设计说明**：司机首次创建时状态为"待审核"，只有活动负责人审核通过后才变为"可调配"进入资源池。审核不通过的司机可修改信息后重新提交审核。已审核通过后设置为"不可用"的司机，恢复时直接变回"可调配"，无需再次审核。活动结束后，关联的所有司机自动置为"不可用"，需调度员手动恢复。

### 3.5 活动时期与状态

活动分为三个时期，不同时期可执行的操作不同：

| 时期 | 说明 | 可执行操作 |
|------|------|------------|
| 筹备期 | 活动创建后的初始阶段，进行资源准备和任务预分配 | 关联/取消关联合供应商、管理车辆/司机（关联/取消关联/审核）、创建任务、编辑任务、分配车辆与司机（预分配）、取消任务 |
| 执行期 | 活动负责人点击"开始活动"后进入，可进行任务派发与调度 | 筹备期所有操作 + 派发任务、改派任务、轨迹追踪、轨迹大屏 |
| 结束期 | 活动收尾归档，关联车辆/司机自动置为不可用 | 查看（只读），不可编辑和派发 |

**时期流转规则：**

```
筹备期 → 执行期（活动负责人点击"开始活动"）
执行期 → 结束期（活动负责人点击"结束活动"，需所有任务办结）
```

- 筹备期 → 执行期：活动负责人点击"开始活动"→ 系统校验是否有关联车辆/司机（至少1辆车+1名司机审核通过）→ 确认后活动进入执行期
- 执行期 → 结束期：活动负责人点击"结束活动"→ 系统校验是否还有未完成任务 → 若有则提示需先处理 → 若无则标记为"已结束" → **系统自动将本次活动关联的所有车辆和司机状态置为"不可用"**
- 边界条件：筹备期不可派发任务，但可创建任务和预分配资源；执行期派发任务时，司机收到通知；活动结束后关联的车辆/司机自动置为"不可用"，不再回到资源池，需调度员手动恢复

### 3.6 供应商状态机

```
合作中 → 已暂停 → 已终止
  ↑        ↓
  └────────┘（可恢复为合作中）
```

| 状态 | 说明 | 对资源分配的影响 |
|------|------|------------------|
| 合作中 | 供应商正常合作中 | 名下车辆/司机可正常分配；可被活动关联 |
| 已暂停 | 合作暂时中止 | 名下车辆/司机分配时系统弹出警告，调度员确认后可继续分配；不可被新活动关联 |
| 已终止 | 合作已终止，终态不可恢复 | 名下车辆/司机禁止分配新任务；已有任务不受影响；不可被新活动关联 |

### 3.7 异常任务判断规则

系统按以下规则自动识别异常任务，被标记为异常的任务在任务列表、活动信息页、移动端看板中均以红色标签突出展示：

| 规则编号 | 异常类型 | 判断条件 | 自动处理 |
|----------|----------|----------|----------|
| E1 | 派发超时 | 任务状态为"待接收"，且派发时间距今超过预设阈值（默认30分钟） | 自动提醒现场调度员 |
| E2 | 拒绝未改派 | 任务状态为"已拒绝"，且拒绝时间距今超过10分钟未改派或取消 | 自动提醒现场调度员和活动总调度员 |
| E3 | 到时未执行 | 任务状态为"待接收"或"已接收"，且当前时间已超过任务计划开始时间 | 自动提醒现场调度员 |
| E4 | 执行超时 | 任务状态为"执行中"，且当前时间已超过任务计划结束时间 | 自动提醒现场调度员 |
| E5 | 司机离线 | 任务状态为"执行中"，但司机GPS位置超过10分钟未更新 | 自动提醒现场调度员 |
| E6 | 故障上报 | 司机主动上报车辆故障（已接收或执行中的任务） | 任务标记为异常，关联车辆状态置为"不可用"，通知现场调度员和活动总调度员 |

> **设计说明**：
> - 异常标记为实时计算，不改变任务本身的状态，仅作为展示层面的标识（E6除外，E6会联动变更车辆状态）
> - 多条异常规则可同时命中同一任务（如E3+E5），列表中显示所有命中的异常标签
> - 异常阈值（30分钟、10分钟等）可在系统管理后台配置
> - 异常任务在活动信息页的统计区独立统计数量并重点列出
> - E6为司机主动触发的异常，与E1~E5系统自动识别不同，上报时司机需填写故障描述

### 3.8 核心业务流程

**正常调度流程：**
1. 调度员创建活动，录入活动名称、时间、地点等信息，指定多位负责人 → 活动进入**筹备期**
2. 调度员在活动详情页"关联供应商"Tab中，从供应商池选择状态为"合作中"的供应商关联至本次活动
3. 调度员在活动详情页"工作小组"Tab中，创建工作小组，维护地点、现场负责人（非必填）、现场调度员
4. 供应商联系人录入车辆和司机信息（活动选择仅限所属供应商已关联的筹备期/执行期活动），上传审核材料，提交审核
5. 活动负责人审核车辆和司机 → 审核通过后进入活动关联资源池（状态为"可调配"）
6. 调度员在活动下创建任务，指定现场调度员，预分配车辆和司机
   - 现场调度员新建任务 → 任务状态为"待审批"，系统根据出发地匹配工作小组 → 有现场负责人则由其审批，无则由总调度员审批 → 审批通过后变为"待派发"
   - 活动总调度员新建任务 → 无需审批，直接进入"待派发"
7. 活动负责人点击"开始活动" → 活动进入**执行期**
8. 调度员派发任务 → 任务状态变为"待接收"，系统通知司机
9. 司机接收/开始执行任务
10. 任务执行中，系统实时记录司机GPS位置，形成行动轨迹；调度员可在任务详情查看轨迹，在轨迹大屏查看全局
11. 司机完成任务 → 车辆和司机自动回收到活动关联池
12. 所有任务办结后，活动负责人点击"结束活动" → 活动进入**结束期** → **系统自动将所有关联车辆和司机置为"不可用"**

**资源审核流程：**
1. 供应商联系人登录系统，在车辆/司机管理页录入信息并上传审核材料（所属活动仅可选择本供应商已关联的筹备期/执行期活动）
2. 车辆/司机状态为"待审核"
3. 活动负责人在审核列表中查看待审核资源及审核材料
4. 审核通过 → 状态变为"可调配"，资源进入活动关联资源池，供应商联系人收到通过通知
5. 审核不通过 → 状态变为"审核不通过"，活动负责人填写驳回原因，供应商联系人收到驳回通知及原因
6. 供应商联系人修改信息后重新提交 → 状态变回"待审核"

**异常任务处理流程：**
1. 系统按异常规则自动识别异常任务，标记并通知对应现场调度员
2. 现场调度员在任务列表或活动信息页发现异常任务
3. 根据异常类型处理：联系司机、改派、取消等
4. 处理后异常标记自动消除（如改派后E2消除、开始执行后E3消除）

**任务审批流程：**
1. 现场调度员在活动下创建任务 → 任务状态为"待审批"
2. 系统根据任务出发地匹配活动中的工作小组地点
3. 若匹配到有现场负责人的工作小组 → 通知该现场负责人审批
4. 若匹配到的工作小组无现场负责人，或未匹配到工作小组 → 通知活动总调度员审批
5. 审批通过 → 任务状态变为"待派发"，可分配车辆和司机
6. 审批不通过 → 填写不通过原因 → 任务状态变为"已取消"，通知任务创建者
7. 活动总调度员新建的任务无需审批，直接进入"待派发"

**司机暂停/继续执行流程：**
1. 司机在执行中点击"暂停任务" → 填写暂停原因（必填）→ 提交暂停申请
2. 任务状态变为"已暂停"，GPS采集暂停
3. 系统通知活动总调度员审核暂停申请
4. 总调度员审核通过 → 暂停确认，GPS保持暂停
5. 总调度员审核不通过 → 任务恢复为"执行中"，GPS恢复采集，通知司机
6. 司机点击"继续执行" → 任务直接恢复为"执行中"，GPS恢复采集（不需审批）

**故障上报流程：**
1. 司机对已接收或执行中的任务点击"故障上报" → 填写故障描述（必填）→ 提交
2. 任务标记为异常（E6），通知现场调度员和活动总调度员
3. 关联车辆状态自动置为"不可用"（防止该车辆被继续分配）
4. 调度员收到通知后跟进处理：安排替代车辆、改派任务等
5. 车辆故障排除后，调度员手动将车辆恢复为"可调配"

**供应商管理流程：**
1. 调度员新增供应商，选择供应商类型，录入资质与联系信息，为供应商联系人开通系统账号
2. 活动创建后，调度员在活动详情页关联供应商
3. 供应商联系人登录系统，录入本公司车辆和司机信息（活动选择受限）
4. 活动负责人审核资源
5. 供应商详情页可查看参与活动、名下所有车辆和司机及其审核状态与使用状态
6. 供应商合作状态变更时，系统自动同步影响范围

**活动结束流程：**
1. 活动负责人确认所有任务已办结
2. 点击"结束活动"→ 系统校验无未完成任务 → 确认
3. 活动进入"结束期"（只读）
4. **系统自动将本次活动关联的所有车辆和司机状态置为"不可用"**
5. 通知所有关联调度员和供应商联系人活动已结束
6. 后续如需复用车辆/司机，调度员在车辆/司机管理页手动恢复为"可调配"

**移动端调度流程：**
1. 调度员在活动现场打开移动端，进入任务页
2. 顶部任务统计区展示各状态任务数量，点击可展开具体任务列表
3. 任务卡片精简展示核心信息，快速操作
4. 处理异常、创建任务、改派等

## 4 功能模块详细说明

### 4.1 活动管理（PC端）

#### 4.1.1 模块概述

活动是任务的容器与上下文，用于将同一批次/同一事件的用车任务组织在一起。调度员可在活动维度下统一管理供应商、车辆、司机和任务。活动需先关联供应商，该供应商名下的车辆/司机才可被添加到本活动。活动分为筹备期、执行期、结束期三个时期，不同时期可执行的操作不同。活动结束后，关联的车辆和司机自动置为不可用。

#### 4.1.2 功能清单

| 功能 | 说明 |
|------|------|
| 创建活动 | 新增一个活动，录入基本信息，指定多位负责人，活动自动进入筹备期 |
| 编辑活动 | 修改活动信息 |
| 开始活动 | 活动负责人将活动从筹备期切换至执行期，允许派发任务 |
| 关联供应商 | 从供应商池中选择状态为"合作中"的供应商关联至本次活动 |
| 取消关联合供应商 | 将已关联的供应商从活动中移除 |
| 管理工作小组 | 在活动下创建、编辑、删除工作小组，维护地点、现场负责人、现场调度员 |
| 关联车辆 | 将车辆池中的可用车辆关联至活动 |
| 关联司机 | 将司机池中的可用司机关联至活动 |
| 取消关联 | 将已关联的车辆/司机从活动中移除 |
| 查看活动详情 | Tab式详情页：活动信息、工作小组、关联供应商、关联车辆、关联司机、任务列表、轨迹大屏 |
| 结束活动 | 活动结束后标记为"已结束"，关联的未完成任务需先处理，结束后车辆/司机自动置为不可用 |
| 活动列表 | 按状态/时期/时间筛选查看所有活动 |

#### 4.1.3 功能详细说明

**创建活动**

- 操作入口：活动管理页 → 新建活动
- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 活动名称 | 文本 | 是 | 如"2026年全市工作会议" |
| 活动时间 | 时间范围 | 是 | 开始时间~结束时间 |
| 活动地点 | 文本 | 否 | 活动举办地点 |
| 活动描述 | 富文本 | 否 | 活动背景、特殊要求等 |
| 负责人 | 多选 | 是 | 关联系统中的调度员，可指定多位。第一位负责人默认为活动创建者。所有负责人均为该活动的总调度员，可查看和操作活动下所有任务 |
| 时期 | 枚举 | 自动 | 默认"筹备期" |

- 交互逻辑：填写表单 → 点击保存 → 系统创建活动（时期为"筹备期"），跳转至活动详情页
- 边界条件：活动时间不可早于当前时间（创建时校验）；同一时间段允许存在多个活动；负责人至少1位

**开始活动（筹备期→执行期）**

- 操作入口：活动详情页 → "开始活动"按钮（仅筹备期显示，仅活动负责人可操作）
- 交互逻辑：
  1. 点击"开始活动"→ 系统校验活动是否有关联的审核通过车辆和司机（至少1辆车+1名司机）
  2. 校验不通过：提示"活动尚未关联足够的可用资源，请先关联供应商并审核通过至少1辆车和1名司机"
  3. 校验通过：弹出确认框"确认开始活动？进入执行期后可派发任务"→ 确认 → 活动时期变为"执行期"
- 边界条件：仅活动负责人可操作；仅筹备期可点击；进入执行期后不可回退到筹备期

**结束活动（执行期→结束期）**

- 操作入口：活动详情页 → "结束活动"按钮（仅执行期显示，仅活动负责人可操作）
- 交互逻辑：
  1. 点击"结束活动"→ 系统校验是否还有未完成任务（待派发/待接收/已接收/执行中）
  2. 若有未完成任务：提示"活动下还有X个未完成任务，请先处理"
  3. 若无未完成任务：弹出确认框"确认结束活动？结束后关联的所有车辆和司机将置为不可用状态"→ 确认 → 活动时期变为"结束期"
  4. **系统自动将本次活动关联的所有车辆和司机状态置为"不可用"**
  5. 系统通知所有关联调度员和供应商联系人"活动{活动名称}已结束"
- 边界条件：仅活动负责人可操作；仅执行期可点击；活动结束后不可恢复到执行期；车辆/司机被置为"不可用"后，如需复用需调度员在车辆/司机管理页手动恢复为"可调配"；若车辆/司机正在执行其他活动的任务（多活动关联），仅取消与本次活动的关联关系，不改变其状态

**活动详情页（Tab式布局）**

活动详情页采用Tab页签布局，共7个Tab：

**Tab 1：活动信息**

- 展示活动基本信息（名称、时间、地点、描述、负责人列表、时期标签）
- 活动时期标识：页面顶部醒目显示当前时期（筹备期/执行期/结束期），不同时期用不同颜色标签
- 筹备期提示语："活动筹备中，可管理资源与创建任务，暂不可派发"
- 执行期提示语："活动执行中，可派发任务与实时调度"
- 操作按钮区（根据时期动态显示）：
  - 筹备期：显示"开始活动"按钮（仅活动负责人可见）
  - 执行期：显示"结束活动"按钮（仅活动负责人可见）
  - 结束期：无操作按钮
- **任务数据统计区**：

| 统计项 | 说明 |
|--------|------|
| 任务总数 | 当前活动下的任务总数 |
| 待派发数 | 状态为"待派发"的任务数量 |
| 进行中数 | 状态为"待接收/已接收/执行中"的任务数量 |
| 已完成数 | 状态为"已完成"的任务数量 |
| 已取消数 | 状态为"已取消"的任务数量 |
| 异常任务数 | 被异常规则命中的任务数量（红色高亮） |
| 异常任务列表 | 列出所有异常任务，每条显示：任务名称、异常类型标签（E1~E5）、司机姓名、持续时间；点击跳转至任务详情 |

- **资源审核待办区**（筹备期和执行期显示）：

| 统计项 | 说明 |
|--------|------|
| 待审核车辆数 | 状态为"待审核"的车辆数量（红色高亮，有值时提示"有新车辆待审核"） |
| 待审核司机数 | 状态为"待审核"的司机数量（红色高亮，有值时提示"有新司机待审核"） |
| 审核快捷入口 | 点击"去审核"跳转至审核列表 |

- 交互逻辑：页面加载时实时计算统计数据；异常任务区和审核待办区自动刷新（间隔可配置，默认60秒）

**Tab 2：工作小组**

- 展示当前活动下所有工作小组列表
- 列表字段：地点、现场负责人（非必填，显示姓名和手机号）、现场调度员（显示姓名和手机号）、待审批任务数
- 操作：
  - **新建工作小组**：点击"新建工作小组"→ 填写地点（文本，必填）、现场负责人（从系统调度员中选择，非必填）、现场调度员（从系统调度员中选择，必填）→ 保存
  - **编辑工作小组**：修改地点、现场负责人、现场调度员
  - **删除工作小组**：删除无待审批任务的工作小组；有待审批任务的不可删除
- 交互逻辑：工作小组按创建时间排序；同一活动下地点不可重复；点击现场负责人/调度员姓名可查看其基本信息
- 边界条件：工作小组中的现场负责人和现场调度员不限于同一人；同一人可在不同工作小组中担任不同角色

**Tab 3：关联供应商**

- 展示当前活动关联的所有供应商列表
- 列表字段：供应商名称、供应商类型、合作状态、名下车辆数、名下司机数、关联时间
- 操作：
  - **关联供应商**：点击"关联供应商"按钮 → 弹窗显示供应商池中状态为"合作中"且未关联本活动的供应商列表 → 勾选后确认 → 关联成功，系统通知供应商联系人
  - **取消关联**：点击"取消关联"→ 确认 → 移除关联。边界条件：若该供应商名下有车辆/司机已被分配到本活动的任务中，不可取消关联，需先处理相关任务
- 筛选条件：按供应商类型、合作状态
- 交互逻辑：供应商列表按关联时间排序；合作状态为"已暂停"或"已终止"的供应商行标黄/标红提示；点击供应商名称可跳转至供应商详情页
- 边界条件：活动关联供应商后，该供应商名下的车辆/司机才可在车辆/司机管理页选择本活动；取消关联供应商后，该供应商名下尚未被分配任务的车辆/司机自动取消与本次活动的关联，已被分配任务的不受影响

**Tab 4：关联车辆**

- 展示当前活动关联的所有车辆列表
- 字段：车牌号、车辆类型、品牌型号、审核状态、当前状态、所属供应商、所属活动、当前任务数
- 操作：关联车辆、取消关联、查看车辆详情、**审核车辆**（待审核状态的车辆显示"审核"按钮）
- 交互逻辑：审核状态为"待审核"的车辆行高亮显示，提供快捷审核入口

**Tab 5：关联司机**

- 展示当前活动关联的所有司机列表
- 字段：姓名、手机号、驾照类型、审核状态、当前状态、所属供应商、所属活动、当前任务数
- 操作：关联司机、取消关联、查看司机详情、**审核司机**（待审核状态的司机显示"审核"按钮）
- 交互逻辑：审核状态为"待审核"的司机行高亮显示，提供快捷审核入口

**Tab 6：任务列表**

- 展示当前活动下所有任务（受调度员权限控制：现场调度员只看自己的任务，总调度员看所有）
- 字段：任务名称、任务类型、开始时间、司机、车辆、现场调度员、状态、异常标签
- **状态角标**：列表顶部展示各状态Tab（全部/待审批/待派发/待接收/已接收/执行中/已暂停/已完成/已取消），每个状态Tab右上角显示对应任务数量角标（红色圆点+数字），点击切换筛选
- 操作：新建任务（筹备期和执行期可用）、查看任务详情、派发（仅执行期）、改派、取消、提醒接收（待接收状态）
- 交互逻辑：支持按状态/现场调度员/异常状态筛选；筹备期已预分配但未派发的任务状态显示"待派发"

**Tab 7：轨迹大屏**

- 嵌入轨迹大屏视图（详见4.9节），自动定位至当前活动
- 仅执行期和结束期可查看实时轨迹（筹备期无执行中任务，显示提示"活动尚未开始，暂无轨迹数据"）

**关联供应商**

- 操作入口：活动详情页 → 关联供应商Tab → 关联供应商
- 交互逻辑：
  1. 点击"关联供应商"→ 弹窗展示供应商池中**状态为"合作中"且未关联本活动**的供应商列表
  2. 列表字段：供应商名称、供应商类型、名下车辆数、名下司机数、合作起止日期
  3. 勾选一个或多个供应商 → 点击"确认关联"→ 关联成功
  4. 系统通知被关联供应商的联系人"您的公司已被关联至活动{活动名称}，请登录系统录入资源"
- 边界条件：仅状态为"合作中"的供应商可被关联；已关联的供应商不可重复关联；筹备期和执行期可关联供应商，结束期不可

**取消关联合供应商**

- 操作入口：活动详情页 → 关联供应商Tab → 取消关联
- 交互逻辑：点击"取消关联"→ 确认 → 移除关联
- 边界条件：
  - 若该供应商名下有车辆/司机已被分配到本活动的任务中，提示"该供应商名下有X辆车/X名司机正在执行任务，不可取消关联"
  - 取消关联后，该供应商名下尚未被分配到本活动任务的车辆/司机自动取消与本次活动的关联
  - 已被分配任务但任务已完成的车辆/司机，取消关联后不影响历史数据

**关联车辆/司机**

- 操作入口：活动详情页 → 关联车辆Tab/关联司机Tab → 关联
- 交互逻辑：弹窗显示当前资源池中"可调配"状态的车辆/司机列表（仅审核通过的，且所属供应商已关联本活动）→ 勾选后确认 → 关联成功
- 边界条件：同一车辆/司机可被关联至多个活动（需所属供应商已关联该活动）；"待审核"和"审核不通过"的资源不可关联

**取消关联**

- 交互逻辑：在活动详情页点击"取消关联"→ 确认 → 移除关联
- 边界条件：若该车辆/司机已被分配到本活动下的某个任务，不可取消关联

### 4.2 供应商管理（PC端）

#### 4.2.1 模块概述

集中管理所有合作供应商的基础信息与合作状态。供应商联系人是供应商方的系统操作人员，负责录入本供应商名下的车辆和司机信息及审核材料。供应商详情页展示该供应商参与的活动、名下车辆和司机信息。

#### 4.2.2 功能清单

| 功能 | 说明 |
|------|------|
| 新增供应商 | 录入供应商基础信息、合作状态，设置供应商联系人账号 |
| 编辑供应商 | 修改供应商信息 |
| 删除供应商 | 删除无效供应商记录 |
| 查看供应商详情 | 查看供应商信息、参与活动、名下车辆与司机列表（含所属活动列）及审核/使用状态 |
| 变更供应商状态 | 调整供应商合作状态 |
| 供应商列表 | 按类型/状态筛选查看所有供应商 |
| 供应商资源统计 | 统计各供应商名下车辆数、司机数、审核通过/待审核/不通过数量、当前可用资源数 |
| 管理供应商联系人 | 为供应商设置/修改联系人账号，重置密码 |

#### 4.2.3 功能详细说明

**新增供应商**

- 操作入口：供应商管理页 → 新增供应商
- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 供应商名称 | 文本 | 是 | 如"XX汽车服务有限公司" |
| 供应商类型 | 枚举 | 是 | 车辆供应商/司机供应商/综合供应商 |
| 统一社会信用代码 | 文本 | 否 | 18位 |
| 联系人姓名 | 文本 | 是 | 供应商联系人姓名 |
| 联系人电话 | 文本 | 是 | 用于开通系统账号，作为登录手机号 |
| 备用联系电话 | 文本 | 否 | — |
| 公司地址 | 文本 | 否 | — |
| 合作开始日期 | 日期 | 是 | — |
| 合作结束日期 | 日期 | 否 | 不填则视为长期合作 |
| 合作状态 | 枚举 | 自动 | 默认"合作中" |
| 合同编号 | 文本 | 否 | — |
| 备注 | 文本 | 否 | — |

- 交互逻辑：填写表单 → 保存 → 系统创建供应商记录 + 自动创建供应商联系人账号（手机号为登录名，初始密码由系统生成，可重置）→ 供应商联系人可登录系统录入资源
- 边界条件：供应商名称不可重复；联系人电话需为有效手机号

**供应商联系人**

- 供应商联系人是供应商方的系统操作人员，通过手机号登录系统
- 权限范围：仅可操作本供应商名下的车辆和司机（增删改查），上传审核材料，查看审核状态；不可操作其他供应商资源，不可进行任何调度操作；可查看本供应商关联的活动信息
- 账号管理：新增供应商时自动创建；联系人电话变更时同步更新登录账号；密码重置由系统管理员操作

**供应商详情页**

供应商详情页采用分区布局，包含以下区域：

- **基本信息区**：展示供应商基础字段

- **参与活动区**：
  - 列表展示该供应商已关联的所有活动
  - 列表字段：活动名称、活动时期（筹备期/执行期/结束期）、活动时间、关联时间、名下车辆数（本次活动下）、名下司机数（本次活动下）
  - 交互逻辑：点击活动名称可跳转至活动详情页；支持按活动时期筛选
  - 边界条件：活动结束后仍保留关联记录（供历史查询）

- **名下车辆区**：
  - 列表展示所有车辆，字段：车牌号、车辆类型、审核状态、当前状态、**所属活动**、关联时间
  - "所属活动"列：展示该车辆关联的活动名称，若关联多个活动则以标签形式展示（如"活动A、活动B"），点击活动名称可跳转
  - 审核状态标签：待审核（黄色）、审核通过（绿色，等同于可调配/已调度/执行中）、审核不通过（红色）

- **名下司机区**：
  - 列表展示所有司机，字段：姓名、手机号、驾照类型、审核状态、当前状态、**所属活动**、关联时间
  - "所属活动"列：展示该司机关联的活动名称，若关联多个活动则以标签形式展示（如"活动A、活动B"），点击活动名称可跳转
  - 审核状态标签同上
  - 操作：查看详情、审核（仅活动负责人可见，对审核不通过的司机显示）

- **资源统计区**：
  - 车辆：总数、审核通过数、待审核数、审核不通过数、当前可用数
  - 司机：总数、审核通过数、待审核数、审核不通过数、当前可用数
  - 参与活动数：当前关联的活动数量

### 4.3 车辆管理池（PC端）

#### 4.3.1 模块概述

集中管理所有可用车辆信息，为活动关联和任务分配提供车辆资源。车辆必须关联所属供应商，新增时必须选择对应活动，**活动选择仅限所属供应商已关联的筹备期/执行期活动**，需上传审核材料，经活动负责人审核通过后方可进入可分配资源池。

#### 4.3.2 功能清单

| 功能 | 说明 |
|------|------|
| 新增车辆 | 录入车辆基础信息，选择所属活动（限所属供应商关联的筹备期/执行期活动），所属供应商锁定为当前操作供应商，上传审核材料 |
| 编辑车辆 | 修改车辆信息（待审核和审核不通过状态可编辑） |
| 删除车辆 | 删除无效车辆记录 |
| 上传审核材料 | 上传车辆照片、年检合格证、保险等审核材料 |
| 审核车辆 | 活动负责人审核车辆，通过或驳回 |
| 绑定默认司机 | 审核通过的车辆可绑定已审核通过的司机作为默认司机，分配车辆时自动带出 |
| 绑定活动地点 | 车辆可绑定活动中的工作小组地点，绑定地点的车辆现场调度员可分配 |
| 批量维护驻点场地 | 批量为多辆车辆绑定活动地点，需满足条件：所选车辆为同一活动且审核通过 |
| 设置不可用 | 标记车辆为维修/停用等不可调配状态 |
| 恢复可用 | 将不可用车辆恢复为可调配 |
| 批量导入 | 通过Excel模板批量导入车辆信息 |
| 下载导入模板 | 下载车辆导入Excel模板 |
| 车辆列表 | 按审核状态/状态/供应商/活动/类型筛选查看 |

#### 4.3.3 功能详细说明

**新增车辆**

- 操作入口：车辆管理页 → 新增车辆；或活动详情页 → 关联车辆Tab → 新增并关联；供应商联系人登录后也可新增（仅限本供应商）
- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 车牌号 | 文本 | 是 | 唯一标识，系统校验不可重复 |
| 车辆类型 | 枚举 | 是 | 轿车/SUV/商务车/中巴/大巴/越野车 |
| 品牌型号 | 文本 | 否 | 如"别克GL8" |
| 核载人数 | 数字 | 否 | 不含司机 |
| 颜色 | 文本 | 否 | — |
| 适用驾照 | 枚举 | 否 | 驾驶该车辆所需的最低驾照类型 |
| 可用日期区间 | 时间范围 | 是 | 默认与活动开始结束时间一致 |
| 所属供应商 | 选择 | 是 | 供应商联系人新增时自动填入本供应商，不可修改；调度员新增时下拉选择（仅显示车辆供应商和综合供应商） |
| 所属活动 | 选择 | 是 | **下拉列表仅显示所属供应商已关联的、且处于筹备期或执行期的活动**，不可多选。从活动详情页新增时自动填入当前活动（需校验当前活动是否属于所选供应商关联的活动） |
| 联系电话 | 文本 | 否 | 车辆负责人/供应商联系方式 |
| 备注 | 文本 | 否 | — |
| **审核材料** | — | — | — |
| 车辆照片 | 图片 | 是 | 至少1张，支持多张（最多5张），格式JPG/PNG，单张不超过10MB |
| 年检合格证 | 图片/文件 | 是 | 支持JPG/PNG/PDF，最多3份，单份不超过10MB |
| 保险单 | 图片/文件 | 是 | 支持JPG/PNG/PDF，最多3份，单份不超过10MB |
| 其他材料 | 图片/文件 | 否 | 支持JPG/PNG/PDF，最多5份，单份不超过10MB |
| 状态 | 枚举 | 自动 | 默认"待审核" |

- 交互逻辑：
  1. 供应商联系人/调度员填写表单 → **先选择所属供应商 → "所属活动"下拉列表联动刷新为该供应商已关联的筹备期/执行期活动** → 上传审核材料 → 点击"提交审核"
  2. 系统校验车牌号唯一性 → 校验供应商类型 → 校验活动是否属于该供应商关联且为筹备期/执行期 → 创建车辆（状态为"待审核"）
  3. 活动负责人在审核列表中查看待审核车辆及审核材料 → 审核通过 → 状态变为"可调配"，自动关联至所选活动 → 审核不通过 → 状态变为"审核不通过"，通知供应商联系人
- 边界条件：同一车牌可用日期重复时提示错误；**所属活动下拉为空时提示"当前供应商暂无关联的筹备期/执行期活动，请先在活动详情页关联供应商"**；新增后需审核通过才进入资源池并关联至所选活动；供应商联系人只能为本供应商新增车辆

**活动选择联动逻辑**：
1. 选择"所属供应商"后，系统查询该供应商已关联的活动列表
2. 筛选条件：活动时期为筹备期或执行期
3. 若供应商未关联任何筹备期/执行期活动，"所属活动"下拉为空，表单下方显示提示信息
4. 切换"所属供应商"时，"所属活动"下拉列表自动重新加载
5. 若从活动详情页进入新增，"所属活动"默认为当前活动，"所属供应商"下拉仅显示已关联本活动的供应商

**审核车辆**

- 操作入口：活动详情页 → 关联车辆Tab → 待审核车辆的"审核"按钮；或车辆管理页 → 待审核列表
- 审核页面展示：
  - 车辆基本信息
  - 审核材料预览（图片可放大查看，PDF可在线预览或下载）
- 操作：
  - 审核通过：点击"通过"→ 车辆状态变为"可调配"，自动关联至所选活动，通知供应商联系人
  - 审核不通过：点击"驳回"→ 必填驳回原因 → 车辆状态变为"审核不通过"，通知供应商联系人（含驳回原因）
- 权限：仅活动负责人（活动总调度员）可审核
- 边界条件：一辆车可关联多个活动，需各活动负责人分别审核

**绑定默认司机**

- 操作入口：车辆详情页 → 绑定默认司机；或车辆管理页 → 车辆列表 → 绑定司机
- 交互逻辑：
  1. 点击"绑定默认司机"→ 弹窗展示已审核通过的司机列表（可按供应商、驾照类型筛选）→ 选择司机 → 确认绑定
  2. 一辆车仅可绑定一名默认司机；一名司机可被多辆车绑定为默认司机
  3. 绑定后，在用车任务中分配该车辆时，系统自动带出绑定的默认司机，调度员可手动调整（更换为其他司机或清除）
  4. 若绑定的司机当前状态为"不可用"，分配时自动带出但提示"该默认司机当前不可用，建议更换"
- 边界条件：仅审核通过的车辆可绑定默认司机；仅审核通过的司机可被绑定；车辆和司机需属于同一活动

**绑定活动地点**

- 操作入口：车辆详情页 → 绑定活动地点；或活动详情页 → 关联车辆Tab → 绑定地点
- 交互逻辑：
  1. 点击"绑定活动地点"→ 弹窗展示当前活动的工作小组地点列表 → 勾选一个或多个地点 → 确认绑定
  2. 一辆车可绑定多个活动地点；一个地点可被多辆车绑定
  3. **权限规则**：绑定过地点的车辆，对应工作小组的现场调度员有权限进行分配；未绑定任何地点的车辆，仅活动总调度员有权限分配
- 边界条件：车辆需先关联至活动，才可绑定该活动的工作小组地点；活动结束期不可修改绑定

**批量维护驻点场地**

- 操作入口：车辆管理页 → 勾选多辆车辆 → 点击"批量维护驻点"按钮
- 维护原则：
  1. 所选车辆必须为**同一活动**的车辆（活动ID相同）
  2. 所选车辆必须为**审核通过**状态
- 交互逻辑：
  1. 在车辆管理列表中勾选多辆车辆
  2. 系统校验所选车辆是否满足维护原则：
     - 若不满足（不同活动或未审核通过）：按钮置灰不可点击，鼠标悬停提示"请选择同一活动且审核通过的车辆"
     - 若满足：按钮高亮可点击
  3. 点击"批量维护驻点"→ 弹窗展示该活动的工作小组地点列表（多选）→ 勾选一个或多个地点 → 确认绑定
  4. 确认后，所选车辆均绑定勾选的地点，覆盖原有绑定关系
- 边界条件：仅活动总调度员可操作；活动结束期不可修改绑定；批量操作成功后自动清空选择

**批量导入车辆**

- 操作入口：车辆管理页 → 批量导入
- 交互逻辑：
  1. 下载导入模板（Excel格式，含字段说明和示例数据）
  2. 按模板格式填写车辆信息
  3. 上传Excel文件 + 批量上传审核材料（打包为zip，按车牌号命名文件夹）
  4. 系统校验数据格式和必填项，**校验所属活动是否属于所属供应商关联的筹备期/执行期活动**
  5. 展示校验结果：成功X条、失败Y条（列出失败原因，如车牌号重复、供应商不存在、活动不属于该供应商关联、活动已结束、缺少审核材料等）
  6. 确认导入 → 成功的记录创建车辆（状态为"待审核"）并关联至指定活动和供应商
- 导入模板字段：车牌号、车辆类型、品牌型号、核载人数、颜色、适用驾照、可用日期起、可用日期止、所属供应商名称、所属活动名称、联系电话、备注
- 边界条件：
  - 单次导入上限500条
  - 同一车牌可用日期重复的记录跳过并在结果中标注
  - 供应商名称不匹配的记录跳过
  - **活动名称不匹配、活动不属于该供应商关联、或活动已结束的记录跳过**
  - 批量导入的车辆状态均为"待审核"，需活动负责人批量审核
  - 导入结果可下载（含失败原因）

### 4.5 资源审核管理（PC端）

#### 4.5.1 模块概述

活动负责人在此模块统一审核供应商联系人提交的车辆和司机资源，审核通过后资源方可进入可分配资源池。

#### 4.5.2 功能清单

| 功能 | 说明 |
|------|------|
| 待审核列表 | 查看待审核的车辆和司机，按提交时间排序 |
| 审核详情 | 查看资源基本信息和审核材料 |
| 审核通过 | 审核通过，资源进入资源池 |
| 审核驳回 | 审核不通过，填写驳回原因，供应商联系人可修改后重新提交 |
| 批量审核 | 批量通过或驳回多条待审核记录 |
| 审核记录 | 查看历史审核记录 |

#### 4.5.3 功能详细说明

**待审核列表**

- 操作入口：PC端左侧菜单 → 资源审核；或活动详情页 → 审核快捷入口
- 列表展示字段：
  - 类型标签：车辆/司机
  - 名称：车牌号或司机姓名
  - 所属供应商
  - 关联活动
  - 提交时间
  - 操作：查看详情/审核
- 筛选条件：按类型（车辆/司机）、按活动、按供应商
- 交互逻辑：默认按提交时间倒序排列；待审核数量在活动信息页统计区实时展示

**审核详情页**

- 展示内容：
  - 资源基本信息（车辆/司机的完整字段）
  - 审核材料：图片可点击放大，PDF可在线预览或下载
- 操作：
  - 通过：点击"审核通过"→ 确认 → 资源状态变为"可调配"，自动关联至所选活动，通知供应商联系人"您提交的车辆/司机已审核通过"
  - 驳回：点击"审核驳回"→ 填写驳回原因（必填）→ 确认 → 资源状态变为"审核不通过"，通知供应商联系人"您提交的车辆/司机审核未通过，原因：{驳回原因}，请修改后重新提交"
- 权限：仅活动负责人（活动总调度员）可操作

**批量审核**

- 操作入口：待审核列表 → 勾选多条记录 → 批量通过/批量驳回
- 批量驳回需统一填写驳回原因
- 交互逻辑：批量操作后逐条执行，展示每条结果

**审核记录**

- 展示所有审核操作记录：资源类型、资源名称、审核结果、驳回原因（如有）、审核人、审核时间
- 支持按时间/结果/审核人筛选

### 4.6 任务管理（PC端）

#### 4.6.1 模块概述

任务是车辆调度的核心业务单元，调度员在此模块创建、分配、派发和跟踪所有用车任务。每个任务指定一名现场调度员，默认为任务创建者。筹备期仅可创建和预分配，执行期才可派发。

#### 4.6.2 功能清单

| 功能 | 说明 |
|------|------|
| 创建任务 | 在活动下新建用车任务，指定现场调度员（筹备期和执行期可用）。现场调度员新建的任务需经审批后方可分配资源 |
| 编辑任务 | 修改未派发的任务信息 |
| 审批任务 | 现场负责人或活动总调度员审批现场调度员创建的任务，审批通过后方可分配车辆和司机 |
| 分配车辆与司机 | 为任务指定车辆和司机（筹备期可预分配），分配车辆时展示车型和核载人数，可筛选；分配车辆时自动带出默认司机 |
| 派发任务 | 将任务派发给司机（**仅执行期可派发**） |
| 改派任务 | 更换任务的车辆或司机 |
| 取消任务 | 取消已创建的任务 |
| 催办任务 | 对"待接收"状态的任务，向司机发送站内通知提醒接收 |
| 任务列表 | 按活动/状态/现场调度员/异常状态筛选查看 |
| 任务详情 | 查看任务完整信息、状态变更记录、执行轨迹 |
| 任务跟踪 | 实时查看执行中任务的状态和位置 |

#### 4.6.3 功能详细说明

**创建任务**

- 操作入口：活动详情页 → 任务列表Tab → 新建任务
- 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属活动 | 自动 | 是 | 从活动详情页创建时自动继承，不可修改 |
| 任务名称 | 文本 | 是 | 如"6月10日张局长接机" |
| 任务类型 | 枚举 | 是 | 接送机/办事出行/会议通勤/考察调研/其他 |
| 任务日期 | 日期 | 是 | — |
| 开始时间 | 时间 | 是 | 用车开始时间 |
| 结束时间 | 时间 | 是 | 预计用车结束时间 |
| 出发地 | 文本 | 是 | — |
| 途经点 | 文本列表 | 否 | 支持添加多个途经点（最多10个），可拖拽排序调整顺序。途经点插入在出发地和目的地之间，构成完整路线 |
| 目的地 | 文本 | 是 | — |
| 乘车人/接口人 | 文本 | 否 | — |
| 乘车人电话 | 文本 | 否 | — |
| 乘车人数 | 数字 | 否 | 用于匹配车辆座位数 |
| 任务描述 | 富文本 | 否 | 特殊要求、路线说明等 |
| 分配车辆 | 选择 | 否 | 创建时可暂不分配；仅显示审核通过的"可调配"车辆 |
| 分配司机 | 选择 | 否 | 创建时可暂不分配；仅显示审核通过的"可调配"司机 |
| 现场调度员 | 选择 | 是 | 默认为当前登录用户（任务创建者），可手动修改为其他调度员。下拉列表显示当前活动的所有关联调度员 |
| 状态 | 枚举 | 自动 | 默认"待派发" |

- 交互逻辑：填写表单 → 保存 → 任务创建成功。**现场调度员创建的任务状态为"待审批"，需经审批通过后方可分配车辆和司机；活动总调度员创建的任务直接进入"待派发"状态**
- 途经点交互：点击"添加途经点"按钮 → 新增一行途经点输入框 → 可继续添加 → 支持拖拽排序调整途经点顺序 → 点击途经点右侧"×"可删除该途经点。路线展示格式：出发地 → 途经点1 → 途经点2 → ... → 目的地
- 边界条件：筹备期和执行期均可创建任务；筹备期创建的任务仅可预分配资源，不可派发；现场调度员必须是系统中已注册的调度员；途经点上限10个，可留空（不添加途经点）

**审批任务**

- 操作入口：待审批任务通知 → 跳转至任务详情 → 审批按钮；或活动详情页 → 任务列表Tab → 待审批状态的任务
- 审批规则：
  - 系统根据任务出发地匹配活动中的工作小组地点
  - 匹配到有现场负责人的工作小组 → 该现场负责人审批
  - 匹配到的工作小组无现场负责人，或未匹配到工作小组 → 活动总调度员审批
  - 活动总调度员创建的任务无需审批
- 审批操作：
  - 审批通过：点击"通过"→ 任务状态变为"待派发"，可分配车辆和司机
  - 审批不通过：点击"不通过"→ 必填不通过原因 → 任务状态变为"已取消"，通知任务创建者
- 交互逻辑：审批人收到App推送通知，点击跳转至任务详情进行审批
- 边界条件：审批不通过的任务不可恢复，需重新创建

**派发任务**

- 前置条件：活动必须处于"执行期"
- 交互逻辑：在任务详情页点击"派发"→ 校验活动时期 → 校验资源分配完整性 → 确认派发 → 任务状态变为"待接收"→ 系统通知司机
- 边界条件：筹备期点击派发按钮时提示"活动尚未开始，请在执行期派发任务"；未分配司机时提示"请先分配司机"

**分配车辆与司机**

- 校验规则与V1.4一致（时间段冲突、可用日期、驾照匹配、供应商状态）
- 分配列表仅显示审核通过的"可调配"资源（车辆/司机）
- 分配列表中标注供应商状态异常的资源
- **车辆分配展示**：车辆列表每行展示车型、核载人数、车牌号、品牌型号、当前状态，支持按车型和核载人数筛选
- **默认司机带出**：选择车辆后，若该车辆已绑定默认司机，系统自动填入默认司机；调度员可手动更换或清除
- **车辆分配权限**：绑定过活动地点的车辆，对应工作小组的现场调度员可分配；未绑定地点的车辆，仅活动总调度员可分配

**任务详情**

- 展示内容：
  - 任务基本信息：名称、类型、时间、路线（出发地 → 途经点1 → 途经点2 → ... → 目的地，无途经点时仅显示出发地→目的地）、乘车人、乘车人电话、任务描述、现场调度员
  - 分配信息：车辆（车牌号、类型、供应商）、司机（姓名、电话、驾照类型、供应商）
  - 异常标签：如命中异常规则，显示异常类型标签（E1~E6）
  - 状态变更时间线：记录每次状态变更的时间、操作人、备注
  - **执行轨迹**：任务状态为"执行中"或"已完成"时，展示地图，显示司机当前位置（执行中）或完整行动轨迹（已完成可回放）
  - 操作按钮：根据当前状态和活动时期动态显示（派发/改派/取消）

**任务列表**

- 展示字段：任务名称、任务类型、开始时间、司机、车辆、现场调度员、状态、异常标签
- 筛选条件：按活动、按状态、按现场调度员、按异常状态（仅异常/仅正常）、按日期
- **排序规则**：
  - 异常任务置顶，并显示浅色红底背景（`bg-red-50`），便于快速识别
  - 司机申请暂停的任务置顶，操作列增加"审核"按钮，点击后弹窗显示暂停申请原因，可进行同意/驳回操作
- 权限控制：现场调度员只能看到自己负责的任务；活动总调度员可看到活动下所有任务

**催办任务**

- 操作入口：任务列表页 → 待接收状态的任务 → "提醒接收"按钮；或任务详情页 → "提醒接收"按钮
- 交互逻辑：
  1. 点击"提醒接收"→ 系统向司机发送站内通知"您有任务待接收：{任务名称}，请及时处理"
  2. 提醒接收按钮点击后有30秒冷却时间，防止频繁提醒
  3. 提醒接收记录在任务详情的状态变更时间线中留痕
- 边界条件：仅"待接收"状态的任务可提醒接收；提醒接收不影响任务状态

### 4.7 司机端 - 任务管理（移动端）

#### 4.7.1 模块概述

司机通过移动端查看和管理自己的任务，是任务执行的唯一入口。任务执行中自动上传GPS位置信息。

#### 4.7.2 导航栏

底部导航栏共2个Tab：

| Tab | 图标 | 说明 |
|-----|------|------|
| 任务 | 📋 | 任务管理页（首页），包含任务统计区和任务列表 |
| 我的 | 👤 | 个人信息、设置 |

> **V1.5变更**：移除原"看板"Tab，任务功能统一集中在"任务"Tab中，通过统计区+列表方式呈现。

#### 4.7.3 功能清单

| 功能 | 说明 |
|------|------|
| 任务统计区 | 顶部展示各状态任务数量，点击可展开对应状态的任务列表 |
| 任务列表 | 按状态分组展示任务（精简卡片） |
| 任务详情 | 查看任务完整信息（含现场调度员姓名和联系方式，可一键拨打） |
| 接收任务 | 确认接收待接收的任务 |
| 拒绝任务 | 拒绝待接收的任务（需填原因） |
| 开始执行 | 开始执行已接收的任务，开启GPS位置上报 |
| 暂停任务 | 执行中因故暂停任务，填写暂停原因，GPS采集暂停 |
| 继续执行 | 恢复执行已暂停的任务，GPS采集恢复（不需审批） |
| 完成任务 | 标记任务为已完成，停止位置上报 |
| 故障上报 | 对已接收/执行中的任务上报车辆故障 |
| 历史任务 | 查看已完成/已取消的历史任务 |
| 告知书确认 | 首次进入时弹窗确认告知书（涉及获取定位信息） |

#### 4.7.4 功能详细说明

**任务页 — 任务统计区**

- 位置：任务页顶部
- 展示内容：横向排列的统计卡片，每个卡片显示一个状态的任务数量

| 统计项 | 颜色 | 说明 |
|--------|------|------|
| 待接收 | 橙色 | 待接收任务数，有值时角标提醒 |
| 已接收 | 蓝色 | 已接收任务数，有值时角标提醒 |
| 执行中 | 绿色 | 执行中任务数，有值时角标提醒 |

- 交互逻辑：
  - 点击任意统计卡片 → 下方列表筛选为对应状态的任务
  - 默认展示"待接收"状态的任务列表（如有待接收任务）
  - 统计数据每次进入页面自动刷新
- **弹窗优化**：所有操作弹窗（拒绝、暂停、故障、完成任务）直接弹出，无需返回主页面

**告知书确认**

- 首次进入司机端时，弹出告知书弹窗，内容涉及获取定位信息：
  - 标题：位置信息采集告知书
  - 正文：为保障用车任务顺利执行，本系统将在任务执行期间采集您的GPS定位信息，用于任务轨迹记录与调度监控。采集范围仅限于任务执行期间，任务完成后自动停止采集。您的位置信息仅用于公务车辆调度管理，不会用于其他用途。相关数据将按规定的期限保存。
  - 按钮：「我已阅读并同意」/「拒绝」
- 交互逻辑：
  - 点击「我已阅读并同意」→ 告知书确认状态记录，不再重复弹出
  - 点击「拒绝」→ 提示"拒绝后部分功能可能受限"，可选择继续拒绝或重新确认
  - 后续可在"我的"→"隐私设置"中重新查看告知书
- 边界条件：告知书仅需确认一次，确认后不再弹出；更换设备或清除数据后需重新确认

**任务页 — 任务列表**

- 位置：任务统计区下方
- **任务卡片UI优化**：

| 展示内容 | 说明 |
|----------|------|
| 状态标签 | 左上角彩色标签（待接收=橙，已接收=蓝，执行中=绿，已完成=灰） |
| 日期时间 | 状态标签右侧，第一行展示：日期 + 开始时间-结束时间 |
| 任务名称 | 第二行，单行，超出省略 |
| 行程时间轴 | 横向时间轴方式展示行程安排：出发地 → 途经点1 → 途经点2 → ... → 目的地，每个地点显示名称和预计到达时间（分上下两行） |
| 到达状态 | 已到达的地点置灰显示并添加删除线 |

- **交互逻辑**：
  - 点击卡片 → 进入任务详情页
  - 点击途经点/出发地/目的地 → 若任务状态为"已接收"或"执行中"且该地点未到达，弹出确认到达弹窗
  - 已到达的地点置灰，不可点击确认到达
  - 根据任务状态显示操作按钮（待接收：接收/拒绝；已接收：开始执行；执行中：完成任务）
- 列表加载：下拉刷新，上拉加载更多

**任务详情**

- 展示内容：
  - 任务基本信息：名称、类型、时间、路线（出发地 → 途经点1 → 途经点2 → ... → 目的地，无途经点时仅显示出发地→目的地）、乘车人、乘车人电话、任务描述
  - 分配信息：车辆（车牌号、类型、供应商）、司机（姓名、电话、驾照类型、供应商）
  - **现场调度员信息**：姓名、手机号，手机号可一键拨打电话
  - 增加"轨迹"区域，可展示司机当前位置（执行中）或完整行动轨迹（已完成可回放）
  - 操作按钮：根据当前状态动态显示（接收/拒绝/开始执行/暂停任务/继续执行/完成任务/故障上报）

**暂停任务**

- 交互逻辑：点击"暂停任务"→ 弹窗填写暂停原因（必填，文本）→ 确认提交 → 任务状态变为"已暂停"，GPS采集暂停 → 系统通知活动总调度员审核
- 边界条件：仅执行中的任务可暂停；暂停后总调度员可审核，审核不通过则任务恢复执行中

**继续执行**

- 交互逻辑：点击"继续执行"→ 任务直接恢复为"执行中"，GPS恢复采集（不需审批）
- 边界条件：仅已暂停的任务可继续执行

**故障上报**

- 交互逻辑：点击"故障上报"→ 弹窗填写故障描述（必填，文本）→ 确认提交 → 任务标记为异常（E6），关联车辆状态置为"不可用" → 通知现场调度员和活动总调度员
- 边界条件：仅已接收和执行中的任务可上报故障；故障上报后车辆不可用，需调度员手动恢复

**开始执行**

- 交互逻辑：点击"开始执行"→ 确认 → 任务状态变为"执行中" → **系统请求GPS定位权限并开始位置上报**（间隔可配置，默认30秒）
- 边界条件：若司机未授权GPS定位，弹出提示"需要开启定位权限以记录任务轨迹"，可继续执行但轨迹数据不完整

**完成任务**

- 交互逻辑：点击"完成"→ 确认 → 任务状态变为"已完成" → **停止GPS位置上报**

**轨迹位置上报规则**

- 上报条件：任务状态为"执行中"时自动上报
- 上报频率：默认30秒/次，可在系统管理后台配置
- 上报内容：经度、纬度、时间戳、速度（如有）、方向（如有）
- 网络异常时：位置信息缓存在本地，网络恢复后自动补传
- 后台运行：司机端App切至后台时继续上报位置（需申请后台定位权限）
- 隐私说明：首次使用时向司机说明轨迹采集目的和范围，司机同意后开始采集

### 4.10 异常任务管理

#### 4.10.1 模块概述

系统按预设规则自动识别异常任务，在多个入口突出展示，帮助调度员快速定位和处理。异常标记不影响任务状态，仅作为展示层面的标识。

#### 4.10.2 异常判断规则

详见第3.7节异常任务判断规则表格。核心规则：
- E1 派发超时：待接收超过30分钟
- E2 拒绝未改派：已拒绝超过10分钟
- E3 到时未执行：已过计划开始时间仍未执行
- E4 执行超时：已过计划结束时间仍在执行
- E5 司机离线：执行中GPS超过10分钟未更新

#### 4.10.3 异常展示入口

| 入口 | 展示方式 |
|------|----------|
| 活动详情 → 活动信息Tab | 统计区显示异常任务数（红色高亮），下方列出异常任务清单 |
| 活动详情 → 关联供应商Tab | 供应商行中标注名下是否有异常任务 |
| 活动详情 → 任务列表Tab | 异常任务行标红，显示异常标签 |
| 任务详情页 | 顶部显示异常标签（红色badge），列出命中的异常规则 |
| 调度员移动端 → 任务统计区 | "异常"统计卡片显示数量，点击展开异常任务列表 |
| 轨迹大屏 | 异常司机用红色图标标注，右侧面板异常分组显示 |

#### 4.10.4 异常处理流程

1. 系统自动识别异常 → 标记并通知现场调度员
2. 调度员从任一入口发现异常任务
3. 根据异常类型采取处理措施：
   - E1/E3：联系司机确认情况，必要时改派
   - E2：立即改派或取消
   - E4：联系司机确认预计完成时间
   - E5：联系司机确认设备状态
4. 处理后异常条件消除 → 标记自动移除

1. App推送为默认方式，短信为补充方式（可配置开关）
2. 超时提醒阈值可配置
3. 供应商合作到期提醒：到期前7天自动触发
4. 异常任务通知：首次命中异常规则时立即通知，之后每30分钟重复通知一次（直到异常消除或任务状态变更）
5. 通知记录可在系统中查询
6. 审核结果通知：审核通过/驳回后即时通知供应商联系人，驳回时同步发送短信
7. 活动结束通知：包含资源状态变更提醒，告知车辆/司机已置为不可用

## 6 非功能性需求

### 6.1 安全性

1. 用户登录采用手机号+验证码或账号密码方式，支持登录态管理
2. 数据传输全程HTTPS加密
3. 操作日志记录：所有关键操作需记录操作人、操作时间、操作内容
4. 角色权限控制：不同角色和调度级别仅可访问其权限范围内的功能和数据
5. 轨迹数据隐私保护：司机轨迹数据仅用于任务执行监控，不得用于其他目的；任务完成后轨迹数据保留期限不超过1年（可配置）
6. 审核材料安全：上传的审核材料（照片、证件）加密存储，仅审核人员和上传者可查看，不得外泄

### 6.2 性能

1. 页面加载时间 ≤ 2秒
2. 任务派发到司机收到通知延迟 ≤ 5秒
3. 轨迹大屏位置更新延迟 ≤ 30秒
4. 系统支持同时在线用户 ≥ 200人
5. 数据库查询响应时间 ≤ 1秒
6. 批量导入：500条车辆/司机数据导入处理时间 ≤ 30秒
7. 审核材料上传：单文件上传时间 ≤ 10秒（10MB以内）
8. 活动结束批量更新资源状态：100个车辆/司机状态更新处理时间 ≤ 10秒

### 6.3 可用性
